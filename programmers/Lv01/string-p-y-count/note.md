# Problem: 문자열 내 p와 y의 개수

### 문제 
문제: 문자열 s에서 p(또는 P)와 y(또는 Y)의 개수가 같으면 true, 다르면 false를 반환하라.
```
대문자와 소문자가 섞여있는 문자열 s가 주어집니다.
s에 'p'의 개수와 'y'의 개수를 비교해 같으면 True, 다르면 False를 return 하는 solution를 완성하세요.
'p', 'y' 모두 하나도 없는 경우는 항상 True를 리턴합니다.
단, 개수를 비교할 때 대문자와 소문자는 구별하지 않습니다.

예를 들어 s가 "pPoooyY"면 true를 return하고 "Pyy"라면 false를 return합니다.
```
- 제한사항:
  - 문자열 s의 길이 : 50 이하의 자연수
  - 문자열 s는 알파벳으로만 이루어져 있다.
    
### 입출력 예  
```
예시 1: 
입력: "pPoooyY"
출력: true 
```
- `"pPoooyY"`에는 `p`의 개수 2개, `y`의 개수 2개로 같으므로 `true`를 return한다.


## 풀이

### 문제 정보
- 플랫폼: 프로그래머스

- 난이도: Lv 01 
- 주제: 문자열, 카운팅
- 문제 핵심:
  - 문자열 순회, 대소문자 처리 (toLowerCase)
- 최초 풀이: 260413
- 수정 버전: 260427

### 문제 이해
- 입력: 알파벳으로만 이루어진 문자열 `s` 
- 문자열 안에 있는 `p`와 `y`의 개수를 비교한다. 
- 대문자 `P`, `Y`도 각각 `p`, `y`와 같은 문자로 처리해야 한다. 
- `p`와 `y`가 하나도 없는 경우에도 두 개수가 모두 0이므로 `true`를 반환한다.
- 반환 형식: 두 문자의 개수가 같으면 `true`, 다르면 `false` 

### 접근법
1. 소문자 변환 후 p/y 개수 각각 카운팅
   - 대소문자를 먼저 통일하면 조건문을 단순하게 작성할 수 있다.
2. 단일 카운터 방식 
   - `p`는 `+1`, `y`는 `-1`로 처리해 최종값이 0인지 확인한다.
3. 대소문자 직접 비교 방식 
   - `toLowerCase()`를 쓰지 않지만 조건식이 길어진다.

### 코드 설계
1. 문자열 `s`를 소문자로 변환한다.
2. `p`의 개수를 저장할 `pCount`, `y`의 개수를 저장할 `yCount`를 0으로 초기화한다.
3. 문자열을 문자 배열로 변환한 뒤 한 글자씩 순회한다.
   - 현재 문자가 `p`이면 `pCount`를 1 증가시킨다.
   - 현재 문자가 `y`이면 `yCount`를 1 증가시킨다.
4. 반복이 끝나면 `pCount == yCount` 결과를 반환한다.


## 풀이

### Solution1: for문 기본 카운팅 방식 (권장) 
- 핵심 포인트
  - `toLowerCase()`: 문자열 → 소문자로 통일
  - p, y 개수 각각 카운팅
- 시간복잡도: O(n)
  - `n`은 문자열 길이
- 공간복잡도: O(n)
  - `toLowerCase()`와 `toCharArray()`로 새 문자열과 문자 배열을 사용한다.

### Solution2: 단일 카운터 방식 
- 핵심 포인트
  - `count` 변수 하나
  - p가 나오면 +1, y가 나오면 -1
  - → count가 0이면 두 문자의 개수가 같음 = true
- 시간복잡도: O(n)
  - `n`은 문자열 길이
- 공간복잡도: O(n)
  - `toLowerCase()`와 `toCharArray()`로 새 문자열과 문자 배열을 사용한다.

### Solution3: 대소문자 직접 비교 방식 
- 핵심 포인트
  - 소문자 변환 없이 `p`, `P`, `y`, `Y`를 직접 비교한다.
- 시간복잡도: O(n)
  - `n`은 문자열 길이
- 공간복잡도: O(n)
  - `toCharArray()`로 문자 배열을 사용한다.
