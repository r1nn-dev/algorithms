# Problem: 없는 숫자 더하기

문제: 0부터 9까지의 숫자 중 numbers에 없는 숫자들의 합을 반환하라.
```text
0부터 9까지의 숫자 중 일부가 들어있는 정수 배열 numbers가 매개변수로 주어집니다.
numbers에서 찾을 수 없는 0부터 9까지의 숫자를 모두 찾아 더한 수를 return 하도록 solution 함수를 완성해주세요.
```
- 제한사항
  - 1 ≤ numbers의 길이 ≤ 9
  - 0 ≤ numbers의 모든 원소 ≤ 9
  - numbers의 모든 원소는 서로 다르다.

### 입력
- 정수 배열 numbers
### 출력 
- 0부터 9까지의 숫자 중 numbers에 없는 숫자들의 합
### 입출력 예시
```
예시 1:
입력: [1,2,3,4,6,7,8,0]
출력: 14 (5 + 9)

예시 2:
입력: [5,8,4,0,6,7,9]
출력: 6 (1 + 2 + 3)
```

## 풀이

### 문제 이해
- 주제: 배열, 집합,
- 문제 핵심: Set 사용 또는 boolean[] 플래그로 존재 여부 확인
- 핵심 조건:
  - 숫자 범위가 0~9로 매우 작고 고정되어 있다.
  - numbers의 길이는 최대 9이므로 시간복잡도 부담이 거의 없다.
  - 존재 여부를 확인하는 방식으로 풀 수도 있고, 전체 합 45에서 주어진 숫자의 합을 빼는 방식으로 풀 수도 있다.

### 접근법
1. 방법1: 전체 합에서 numbers의 합을 빼는 방식 (권장)
   - 0 + 1 + 2 + ... + 9 = 45
   - 전체 합 45에서 numbers의 모든 원소 합을 빼면 없는 숫자들의 합만 남는다.
   - 이 문제에서는 원소가 중복되지 않기 때문에 직관적으로 푸는게 가장 효율적인 방식이다. 
2. 방법2: boolean[] 플래그 배열로 존재 여부를 확인하는 방식
   - 크기 10짜리 boolean[] exists 배열을 만든다.
   - 0부터 9까지 순회하면서 exists[i] == false인 숫자만 합산한다.
   - 숫자 범위가 넓어지거나, 실제로 누락된 숫자 목록까지 구해야 하는 문제일 경우 더 효과적인 방식이다. 

### 코드 설계
1. 입력: 정수 배열 `numbers`가 주어진다.
2. 0부터 9까지 순회하면서 false인 숫자만 더한다.
3. 출력: 없는 숫자들의 합을 int형로 반환한다.

## 코드 구현
### Solution1: 전체 합에서 numbers의 합을 빼는 방식 (권장)
- 시간복잡도: `O(N)`
- 공간복잡도: `O(1)`

### Solution2: boolean[] 플래그 배열로 존재 여부를 확인하는 방식
- 시간복잡도: O(N + 10) → `O(N)`
- 공간복잡도: O(10) → `O(1)`
