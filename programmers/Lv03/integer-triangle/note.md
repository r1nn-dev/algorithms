# Problem: Title of Problem

### 문제
문제: 삼각형의 맨 위에서 맨 아래까지 가는 경로 중 최대 합을 구하세요.
```text
    7
   3 8
  5 1 0
 2 7 4 4
4 5 2 6 5

위와 같은 삼각형의 꼭대기에서 바닥까지 이어지는 경로 중, 거쳐간 숫자의 합이 가장 큰 경우를 찾아보려고 합니다.
아래 칸으로 이동할 때는 대각선 방향으로 한 칸 오른쪽 또는 왼쪽으로만 이동 가능합니다.
예를 들어 3에서는 그 아래칸의 8 또는 1로만 이동이 가능합니다.

삼각형의 정보가 담긴 배열 triangle이 매개변수로 주어질 때, 거쳐간 숫자의 최댓값을 return 하도록 solution 함수를 완성하세요.
```
- 제한사항
  - 삼각형의 높이는 1 이상 500 이하이다. 
  - 삼각형을 이루고 있는 숫자는 0 이상 9,999 이하의 정수이다. 

### 입력
- 2차원 정수 배열 `triangle`
### 출력 
- 꼭대기에서 바닥까지 이동하면서 얻을 수 있는 최대 합
### 입출력 예시
```
예시 1:
입력:
triangle = [[7],[3,8],[8,1,0],[2,7,4,4],[4,5,2,6,5]]
출력: 30 (7+8+1+5+9)
```


## 풀이

### 문제 정보
- 플랫폼: 프로그래머스 
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/43105
- 난이도: Lv 03 
- 주제: DP, 경로 최적화
- 문제 핵심:
  - DP로 각 위치에서의 최대 합 계산
  - 상향식 DP (아래에서 위로) 
- 최초 풀이: 260516
- 수정 버전: 260519

### 문제 이해
- 입력: 삼각형 형태의 2차원 정수 배열 `triangle`이 주어진다.
- 해야 할 일:
  - 맨 위 칸에서 시작해 맨 아래 줄까지 이동한다.
  - 한 칸 아래로 내려갈 때는 바로 아래 왼쪽 또는 바로 아래 오른쪽으로만 이동할 수 있다.
  - 가능한 모든 경로 중 숫자 합이 가장 큰 값을 구한다.
- 주의할 점:
  - 단순 DFS로 모든 경로를 탐색하면 경우의 수가 너무 많아진다.
  - 삼각형의 높이가 최대 500이므로 완전 탐색은 불가능하다.
  - 각 칸까지 도달하는 최대 합 또는 각 칸에서 바닥까지 갈 수 있는 최대 합을 저장해야 한다.
  - 같은 위치의 결과를 반복 계산하지 않도록 DP를 사용한다.
- 반환 형식: 최대 경로 합을 담은 `int`

### 접근법
1. Solution1: 상향식 DP 방식 
   - 아래쪽 행부터 위로 올라오면서 각 칸에서 선택할 수 있는 두 자식 중 더 큰 값을 더한다.
   - `triangle[i][j]`는 아래 행의 `triangle[i + 1][j]`, `triangle[i + 1][j + 1]` 중 큰 값을 더해 갱신한다.
   - 마지막에는 `triangle[0][0]`에 전체 최대 경로 합이 저장된다.
   - 별도의 DP 배열 없이 기존 `triangle` 배열을 직접 갱신할 수 있어 공간 효율이 좋다.
2. Solution2: 하향식 DP 방식
   - 위에서 아래로 내려가면서 각 칸까지 도달할 수 있는 최대 합을 계산한다.
   - 현재 칸은 위쪽 왼쪽 부모 또는 위쪽 오른쪽 부모에서 올 수 있다.
   - 가장 마지막 행에 도달한 뒤, 마지막 행에서 가장 큰 값을 반환한다.
   - 원본 배열을 보존하고 싶을 때 별도의 `dp` 배열을 사용하는 방식이다.

### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 삼각형의 높이 `n`을 구한다.
2. 초기화할 변수
   - `int n = triangle.length;`
   - Solution1에서는 별도의 배열 없이 `triangle`을 DP 배열처럼 사용한다.
   - Solution2에서는 `int[][] dp = new int[n][n];`을 생성한다.
3. 반복문 안에서 할 일
   - Solution1 기준:
     - 아래에서 두 번째 행부터 첫 번째 행까지 거꾸로 올라간다.
     - 현재 위치 `triangle[i][j]`에 아래쪽 두 칸 중 더 큰 값을 더한다.
     - 갱신식은 다음과 같다.
       - `triangle[i][j] += Math.max(triangle[i + 1][j], triangle[i + 1][j + 1]);`
   - Solution2 기준:
     - 위에서 아래로 내려가며 각 칸까지의 최대 합을 계산한다.
     - 왼쪽 끝 칸은 바로 위의 왼쪽 끝에서만 올 수 있다.
     - 오른쪽 끝 칸은 바로 위의 오른쪽 끝에서만 올 수 있다.
     - 가운데 칸은 위쪽 왼쪽 부모와 위쪽 오른쪽 부모 중 더 큰 값을 선택한다.
4. 반환 형식
   - Solution1은 `triangle[0][0]`을 반환한다.
   - Solution2는 마지막 행의 최댓값을 반환한다.

## 코드 구현

### Solution1: 상향식 DP 방식 (권장)
- 핵심 포인트
  - 바닥 행부터 위로 올라오면서 각 칸에 “이 칸에서 시작했을 때 얻을 수 있는 최대 합”을 저장한다.
  - 현재 칸은 아래쪽의 두 칸 중 더 큰 경로를 선택하면 된다.
  - 기존 `triangle` 배열을 직접 갱신하므로 별도의 DP 배열이 필요 없다.
  - 단, 원본 배열이 변경된다는 점은 주의해야 한다.
- 시간복잡도: O(N<sup>2</sup>)
  - 삼각형의 모든 칸을 한 번씩 확인한다.
- 공간복잡도: O(1)
  - 입력 배열을 직접 DP 배열처럼 사용한다.

### Solution2: 하향식 DP 방식
- 핵심 포인트
  - 꼭대기에서부터 각 칸까지 도달할 수 있는 최대 합을 저장한다.
  - 현재 칸으로 올 수 있는 부모는 최대 2개이다.
  - 왼쪽 끝과 오른쪽 끝은 부모가 1개뿐이므로 따로 처리해야 한다.
  - 원본 `triangle` 배열을 변경하지 않는다.
- 시간복잡도: O(N<sup>2</sup>)
  - 삼각형의 모든 칸을 한 번씩 확인한다.
- 공간복잡도: O(N<sup>2</sup>)
  - 별도의 `dp` 배열을 사용한다.
