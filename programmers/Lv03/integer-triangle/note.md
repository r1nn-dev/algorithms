# Problem: Title of Problem

### 문제
문제: 삼각형의 맨 위에서 맨 아래까지 가는 경로 중 최대 합을 구하세요.
```text
    7
   3 8
  5 1 0
 2 7 4 4
4 5 2 6 5

위와 같은 삼각형의 꼭대기에서 바닥까지 이어지는 경로 중, 거쳐간 숫자의 합이 가장 큰 경우를 찾아보려고 합니다.
아래 칸으로 이동할 때는 대각선 방향으로 한 칸 오른쪽 또는 왼쪽으로만 이동 가능합니다.
예를 들어 3에서는 그 아래칸의 8 또는 1로만 이동이 가능합니다.

삼각형의 정보가 담긴 배열 triangle이 매개변수로 주어질 때, 거쳐간 숫자의 최댓값을 return 하도록 solution 함수를 완성하세요.
```
- 제한사항
  - 삼각형의 높이는 1 이상 500 이하이다. 
  - 삼각형을 이루고 있는 숫자는 0 이상 9,999 이하의 정수이다. 

### 입력
- 삼각형 형태의 2차원 정수 배열 `triangle`
### 출력 
- 꼭대기에서 바닥까지 이동하면서 얻을 수 있는 최대 합
### 입출력 예시
```
예시 1:
입력:
triangle = [[7],[3,8],[8,1,0],[2,7,4,4],[4,5,2,6,5]]
출력: 30 (7+8+1+5+9)
```

- 문제 핵심:

  - 상향식 DP (아래에서 위로) 
- 최초 풀이: 260516

### 문제 이해
- 입력: 삼각형 형태의 2차원 정수 배열 `triangle` 
- 해야 할 일:
  - 맨 위 칸에서 시작한다.
  - 한 줄씩 아래로 내려가며 맨 아래 줄까지 이동한다.
  - 이동할 때는 현재 위치 기준으로 바로 아래 왼쪽 또는 바로 아래 오른쪽으로만 이동할 수 있다.
  - 가능한 모든 경로 중 숫자 합이 가장 큰 값을 구한다.
- 주의할 점:
  - 단순 DFS로 모든 경로를 탐색하면 경우의 수가 매우 빠르게 증가한다.
  - 삼각형의 높이가 최대 500이므로 완전탐색은 부적절하다.
  - 각 위치까지의 최대 합 또는 각 위치에서 바닥까지의 최대 합을 저장해야 한다.
  - DP를 사용하면 같은 위치의 결과를 반복 계산하지 않을 수 있다.
- 반환 형식: 최대 경로 합을 int형으로 반환한다.

### 접근법
1. Solution1: 상향식 DP 방식 (권장)
   - 아래쪽 행부터 위로 올라오면서 각 칸에서 선택할 수 있는 두 자식 중 더 큰 값을 현재 칸에 더한다.
   - 기존 `triangle` 배열을 직접 갱신하므로 별도의 DP 배열이 필요 없다.
   - 단, 원본 배열이 변경된다는 점은 주의해야 한다.
2. Solution2: 하향식 1차원 DP 방식 
   - 위에서 아래로 내려가면서 각 칸까지 도달할 수 있는 최대 합을 1차원 배열에 저장한다.
   - 원본 `triangle` 배열을 변경하지 않으면서도 공간복잡도를 O(N)으로 줄일 수 있다.
   - 한계: 오른쪽에서 왼쪽으로 갱신해야 하므로 인덱스 처리에 주의해야 한다.


### Solution1: 상향식 DP 방식 (권장)
### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 삼각형의 높이 `n`을 구한다.
2. 초기화할 변수
   - `int n = triangle.length;`
   - 별도의 DP 배열 없이 triangle 배열 자체를 DP 저장소로 사용한다.
3. 반복문 안에서 할 일
   - 아래에서 두 번째 행부터 첫 번째 행까지 거꾸로 올라간다.
   - 현재 위치 `triangle[i][j]`에서 선택 가능한 아래쪽 두 칸을 확인한다.
   - 조건에 따른 갱신
     - 왼쪽 아래 값: `triangle[i + 1][j]`
     - 오른쪽 아래 값: `triangle[i + 1][j + 1]`
     - 두 값 중 더 큰 값을 현재 칸에 더한다.
 - 누적: `triangle[i][j]`에는 현재 칸에서 시작해 맨 아래까지 내려갔을 때 얻을 수 있는 최대 합이 저장된다.
 - 비교: `Math.max(triangle[i + 1][j], triangle[i + 1][j + 1])`로 더 큰 경로를 선택한다.
4. 반환 형식
   - 최종적으로 `triangle[0][0]`에 전체 최대 경로 합이 저장되므로 이를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N<sup>2</sup>)`
  - N은 삼각형의 높이이다.
  - 삼각형의 모든 칸을 한 번씩 확인한다.
- 공간복잡도: `O(1)`
  - 입력 배열을 직접 DP 배열처럼 사용한다.

### Solution2: 하향식 DP 방식
- 핵심 포인트
  - 꼭대기에서부터 각 칸까지 도달할 수 있는 최대 합을 저장한다.
  - 현재 칸으로 올 수 있는 부모는 최대 2개이다.
  - 왼쪽 끝과 오른쪽 끝은 부모가 1개뿐이므로 따로 처리해야 한다.
  - 원본 `triangle` 배열을 변경하지 않는다.
- 시간복잡도: O(N<sup>2</sup>)
  - 삼각형의 모든 칸을 한 번씩 확인한다.
- 공간복잡도: O(N<sup>2</sup>)
  - 별도의 `dp` 배열을 사용한다.
