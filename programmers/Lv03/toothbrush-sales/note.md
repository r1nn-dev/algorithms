# Problem: Multi-level Toothbrush Sales

### 문제
문제: 판매원의 상위자에게 수익을 배분하는 시스템에서 각자의 순수 이익을 구하세요.
```text
판매원이 칫솔을 판매하면 그 이익이 피라미드 조직을 타고 조금씩 분배되는 형태의 판매망입니다.
각각은 자신을 조직에 참여시킨 추천인에 연결되어 피라미드 식의 구조를 이루고 있습니다.
조직의 이익 분배 규칙은 간단합니다.
모든 판매원은 칫솔의 판매에 의하여 발생하는 이익에서 10% 를 계산하여 자신을 조직에 참여시킨 추천인에게 배분하고 나머지는 자신이 가집니다.
모든 판매원은 자신이 칫솔 판매에서 발생한 이익 뿐만 아니라,
자신이 조직에 추천하여 가입시킨 판매원에게서 발생하는 이익의 10% 까지 자신에 이익이 됩니다.
자신에게 발생하는 이익 또한 마찬가지의 규칙으로 자신의 추천인에게 분배됩니다.
단, 10% 를 계산할 때에는 원 단위에서 절사하며, 10%를 계산한 금액이 1 원 미만인 경우에는 이득을 분배하지 않고 자신이 모두 가집니다.
```
- 제한사항
  - enroll의 길이는 1 이상 10,000 이하다.
  - referral의 길이는 enroll의 길이와 같다.
  - seller의 길이는 1 이상 100,000 이하이다.
  - amount의 길이는 seller의 길이와 같다.
  - 칫솔 한 개를 판매하여 얻어지는 이익은 100 원으로 정해져 있다.
  - 모든 조직 구성원들의 이름은 10 글자 이내의 영문 알파벳 소문자들로만 이루어져 있다.

### 입력
- `String[] enroll`: 판매원 이름 배열
- `String[] referral`: 각 판매원을 추천한 사람의 이름 배열
- `String[] seller`: 판매 기록이 있는 판매원 이름 배열
- `int[] amount`: 각 판매 기록에서 판매한 칫솔 개수 배열
### 출력 
- enroll 배열의 순서대로 각 판매원이 얻은 최종 이익을 담은 int[] 배열
### 입출력 예시
```
예시 1:
입력:
enroll = ["john", "mary", "edward", "sam", "emily", "jaimie", "tod", "young"]
referral = ["-", "-", "mary", "edward", "mary", "mary", "jaimie", "edward"]
seller = ["young", "john", "tod", "emily", "mary"]
amount = [12, 4, 2, 5, 10]
출력:
result = [360, 958, 108, 0, 450, 18, 180, 1080]

예시 2:
입력:
enroll = ["john", "mary", "edward", "sam", "emily", "jaimie", "tod", "young"]
referral = ["-", "-", "mary", "edward", "mary", "mary", "jaimie", "edward"]
seller = ["sam", "emily", "jaimie", "edward"]
amount = [2, 3, 5, 4]
출력:
result = [0, 110, 378, 180, 270, 450, 0, 0]
```


## 풀이

### 문제 정보
- 플랫폼: 프로그래머스
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/77486 
- 난이도: Lv 03 
- 주제: 그래프, DFS, 트리 구조  
- 문제 핵심:
  - 그래프를 트리로 해석 (상하관계)
  - DFS로 상위자 추적
- 최초 풀이: 260616 
- 수정 버전: 260702 

### 문제 이해
- 입력:
  - `String[] enroll`: 판매원 이름 배열
  - `String[] referral`: 각 판매원을 추천한 사람의 이름 배열
  - `String[] seller`: 판매 기록이 있는 판매원 이름 배열
  - `int[] amount`: 각 판매 기록에서 판매한 칫솔 개수 배열
- 해야 할 일:
  - 각 판매 기록마다 수익을 계산한다.
  - 판매자부터 추천인 방향으로 10%씩 수익을 분배한다.
  - 각 판매원이 최종적으로 얻은 순수 이익을 계산한다.
- 주의할 점:
  - 칫솔 1개당 이익은 100원이다.
  - 현재 사람이 받은 금액의 10%를 추천인에게 넘긴다.
  - 10% 계산은 정수 나눗셈으로 처리하므로 원 단위 절사된다.
  - 넘길 금액이 0원이 되면 더 이상 상위로 전파하지 않는다.
  - `referral` 값이 `"-"`이면 추천인이 없는 루트이다.
  - 같은 판매자가 여러 번 판매할 수 있으므로 판매 기록마다 수익 전파를 수행해야 한다.
- 반환 형식: enroll 배열 순서대로 각 판매원의 최종 이익을 담은 int[] 배열을 반환한다.

### 접근법
1. Solution1: 부모 포인터 + 반복 전파 방식 
   - 판매원 이름을 인덱스로 변환하고, 각 판매자의 추천인을 부모 인덱스로 저장한다.
   - 판매가 발생할 때마다 현재 판매자부터 부모 방향으로 수익을 전파한다.
   - 재귀 DFS를 사용하지 않아 스택 오버플로우 위험이 없고, 코딩테스트에서 가장 안정적이다.


## Solution1: 부모 포인터 + 반복 전파 방식 
### 코드 설계
1. 실제 코드의 첫 번째 동작: enroll 배열을 순회하면서 판매원 이름과 인덱스를 HashMap에 저장한다.
2. 추천 관계를 저장할 parent 배열을 만든다.
   - `parent[i]`: i번 판매원의 추천인 인덱스
   - 추천인이 없으면 -1
3. 최종 이익을 저장할 answer 배열을 만든다.
4. seller 배열을 순회하면서 각 판매 기록을 처리한다.
   - 판매 금액은 `amount[i] * 100`이다.
   - 현재 판매자부터 시작해 추천인 방향으로 이동한다.
   - 현재 금액의 10%를 추천인에게 넘긴다.
   - 현재 사람은 나머지 금액을 가진다.
   - 넘길 금액이 0원이 되면 전파를 종료한다.
5. 반환 형식: `answer` 배열은 `enroll` 순서와 동일하게 구성되어 있으므로 그대로 반환한다.

### 코드 구현 
- 핵심 포인트
- 시간복잡도: `O(N + M × H)`
  - 판매원 수를 N, 판매 기록 수를 M, 추천 관계의 최대 높이를 H라고 하면 O(N + M × H)이다.
  - 단, 전달 금액이 계속 10분의 1로 줄어들기 때문에 실제 반복 횟수는 매우 작다.
- 공간복잡도: `O(N)`
  - HashMap, parent, answer 배열에 판매원 수만큼 공간이 필요하다.
