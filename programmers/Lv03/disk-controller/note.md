# Problem: Disk Controller

### 문제
문제: 작업이 요청될 때마다 대기 시간을 최소화하도록 스케줄링하세요.
```text
하드디스크는 한 번에 하나의 작업만 수행할 수 있습니다. 디스크 컨트롤러를 구현하는 방법은 여러 가지가 있습니다.
이 문제에서는 우선순위 디스크 컨트롤러라는 가상의 장치를 이용한다고 가정합니다.

우선순위 디스크 컨트롤러는 다음과 같이 동작합니다.
    1.
    어떤 작업 요청이 들어왔을 때 작업의 번호, 작업의 요청 시각, 작업의 소요 시간을 저장해 두는 대기 큐가 있습니다.
    처음에 이 큐는 비어있습니다.
    2.
    디스크 컨트롤러는 하드디스크가 작업을 하고 있지 않고 대기 큐가 비어있지 않다면
    가장 우선순위가 높은 작업을 대기 큐에서 꺼내서 하드디스크에 그 작업을 시킵니다.
    이때, 작업의 소요시간이 짧은 것, 작업의 요청 시각이 빠른 것, 작업의 번호가 작은 것 순으로 우선순위가 높습니다.
    3. 
    하드디스크는 작업을 한 번 시작하면 작업을 마칠 때까지 그 작업만 수행합니다.
    4. 
    하드디스크가 어떤 작업을 마치는 시점과 다른 작업 요청이 들어오는 시점이 겹친다면
    하드디스크가 작업을 마치자마자 디스크 컨트롤러는 요청이 들어온 작업을 대기 큐에 저장한 뒤
    우선순위가 높은 작업을 대기 큐에서 꺼내서 하드디스크에 그 작업을 시킵니다.
또, 하드디스크가 어떤 작업을 마치는 시점에 다른 작업이 들어오지 않더라도 그 작업을 마치자마자 또 다른 작업을 시작할 수 있습니다.
이 과정에서 걸리는 시간은 없다고 가정합니다.

각 작업에 대해 [작업이 요청되는 시점, 작업의 소요시간]을 담은 2차원 정수 배열 jobs가 매개변수로 주어질 때, 
우선순위 디스크 컨트롤러가 이 작업을 처리했을 때 모든 요청 작업의 반환 시간의 평균의 정수부분을 return 하는 solution 함수 작성하시오.
```
- 제한사항
  - 1 ≤ jobs의 길이 ≤ 500
  - jobs[i]는 i번 작업에 대한 정보이고 [s, l] 형태이다.
    - 0 ≤ s=작업이 요청되는 시점 ≤ 1,000이다.
    - 1 ≤ l=작업 소요시간 ≤ 1,000입니다.

### 입력
- `jobs[i] = [s, l]`
- `s`: i번 작업의 요청 시각
- `l`: i번 작업의 소요 시간
### 출력 
- 모든 작업의 반환 시간 평균의 정수 부분
### 입출력 예시
```
예시 1:
입력: jobs = [[0, 3], [1, 9], [3, 5]]
출력: 8 (평균 대기 시간)

예시 2:
입력: jobs = [[0, 3], [1, 9], [2, 6]]
출력: 9 (평균 대기 시간)
```

- 주제: 우선순위 큐, 스케줄링
- 문제 핵심:
  - PriorityQueue로 최소 작업 시간 선택 (SJF)
  - 현재 시 기준으로 가능한 작업 중 선택
- 최초 풀이: 260509
- 수정 버전: 260703 

### 문제 이해
- 입력:
  - `jobs[i] = [s, l]`
  - `s`: i번 작업의 요청 시각
  - `l`: i번 작업의 소요 시간
- 해야 할 일:
  - 하드디스크는 한 번에 하나의 작업만 처리한다.
  - 현재 시각까지 요청된 작업 중에서 우선순위가 가장 높은 작업을 선택한다.
  - 모든 작업의 반환 시간을 구한 뒤 평균의 정수 부분을 반환한다.
- 우선순위:
  1. 소요 시간이 짧은 작업
  2. 요청 시각이 빠른 작업
  3. 작업 번호가 작은 작업
- 주의할 점:
  - 아직 요청되지 않은 작업은 선택할 수 없다.
  - 대기 큐가 비어 있는데 아직 처리하지 않은 작업이 남아 있다면, 다음 작업 요청 시각으로 현재 시각을 이동해야 한다.
  - 평균 대기 시간이 아니라 평균 반환 시간을 구해야 한다.
  - 반환 시간은 작업 종료 시각 - 작업 요청 시각이다.
- 반환 형식: 모든 작업의 반환 시간 평균의 정수 부분을 `int`형으로 반환한다.

### 접근법
1. Solutions1: 정렬 + PriorityQueue
   - 작업을 요청 시각 기준으로 정렬한 뒤, 현재 시각까지 요청된 작업을 PriorityQueue에 넣고 소요 시간이 가장 짧은 작업부터 처리한다.
2. Solution2: 전체 탐색 방식
   - 매 시점마다 아직 처리하지 않은 작업 전체를 탐색해 현재 처리 가능한 작업 중 우선순위가 가장 높은 작업을 선택한다.


## Solution1: 정렬 + PriorityQueue 방식 (권장)
### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 작업 번호, 요청 시각, 소요 시간을 저장할 `Job` 클래스를 만든다.
   - `jobs` 배열을 `Job` 배열로 변환한다.
   - `Job` 배열을 요청 시각 기준으로 정렬한다.
2. 초기화할 변수
   - `int n = jobs.length`
   - `Job[] jobList = new Job[n]`
   - `PriorityQueue<Job> pq`
   - `int time = 0`: 현재 시각
   - `int index = 0`: 아직 PriorityQueue에 넣지 않은 작업의 위치
   - `int count = 0`: 처리 완료한 작업 수
   - `int totalTime = 0`: 반환 시간 누적합
3. 반복문 안에서 할 일
   - 모든 작업을 처리할 때까지 반복한다.
   - 현재 시각 이하로 요청된 작업을 PriorityQueue에 넣는다.
   - PriorityQueue가 비어 있지 않으면 우선순위가 가장 높은 작업을 꺼낸다.
   - 조건에 따른 갱신
     - 작업을 처리한 만큼 `time`을 증가시킨다.
     - 반환 시간인 `time - requestTime`을 `totalTime`에 누적한다.
     - 처리 완료한 작업 수 `count`를 증가시킨다.
     - 누적: `totalTime += time - current.requestTime`
     - 비교: PriorityQueue는 소요 시간, 요청 시각, 작업 번호 순으로 우선순위를 비교한다.
     - PriorityQueue가 비어 있다면 다음 작업 요청 시각으로 `time`을 이동한다.
4. 반환 형식: `totalTime / n`을 반환한다.

## 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N log N)`
  - 작업 정렬에 O(N log N)이 걸린다.
  - 각 작업은 PriorityQueue에 한 번 들어가고 한 번 나오므로 O(N log N)이 걸린다.
- 공간복잡도: `O(N)`
  - 작업 배열과 PriorityQueue를 사용한다. 

## Solution2: 전체 탐색 방식
### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 각 작업의 처리 여부를 저장할 visited 배열을 만든다.
2. 초기화할 변수
   - `int n = jobs.length`
   - `boolean[] visited = new boolean[n]`
   - `int time = 0`: 현재 시각
   - `int count = 0`: 처리 완료한 작업 수
   - `int totalTime = 0`: 반환 시간 누적합
3. 반복문 안에서 할 일
   - 모든 작업을 처리할 때까지 반복한다.
   - 아직 처리하지 않은 작업 전체를 순회한다.
   - 현재 시각까지 요청된 작업 중 우선순위가 가장 높은 작업을 선택한다.
   - 조건에 따른 갱신
     - 처리 가능한 작업이 있으면 해당 작업을 수행한다.
     - 처리 가능한 작업이 없으면 아직 처리하지 않은 작업 중 가장 빠른 요청 시각으로 time을 이동한다.
     - 누적: 작업을 처리한 뒤 `totalTime += time - jobs[selected][0]`을 수행한다.
     - 비교:
         - 소요 시간이 더 짧은 작업을 선택한다.
         - 소요 시간이 같다면 요청 시각이 더 빠른 작업을 선택한다.
         - 소요 시간과 요청 시각이 모두 같다면 작업 번호가 작은 작업을 선택한다.
4. 반환 형식: `totalTime / n`을 반환한다.

## 코드 구현
- 핵심 포인트 
- 시간복잡도: O(N<sup>2</sup>)
  - 작업을 하나 처리할 때마다 전체 작업을 탐색할 수 있다.   
- 공간복잡도: `O(N)`
    - 처리 여부를 저장하는 `visited` 배열을 사용한다.
