# Problem: Island Perimeter

### 문제
문제: 
```text
0과 1로 이루어진 2차원 정수 배열 grid가 주어집니다.
grid[i][j]가 1이면 해당 칸은 육지를 의미하고, 0이면 물을 의미합니다.  
육지는 상하좌우로 인접한 경우에만 서로 연결된 것으로 봅니다.
대각선 방향으로 맞닿아 있는 칸은 연결된 것으로 보지 않습니다.

각 칸은 한 변의 길이가 1인 정사각형이며, 격자 바깥은 모두 물로 둘러싸여 있다고 가정합니다.

grid에 표시된 하나의 섬에 대해, 섬의 둘레 길이를 return 하도록 solution 함수를 완성해주세요.
```
- 제한사항
  - 1 <= grid.length <= 100
  - 1 <= grid[i].length <= 100
  - grid[i][j]는 0 또는 1이다.
  - grid에는 하나 이상의 육지 칸이 존재한다.
  - 모든 육지 칸은 상하좌우 방향으로 연결되어 하나의 섬을 이룬다.
  - 섬 내부에는 물로 이루어진 호수가 없다.
  - 대각선 방향으로 인접한 육지는 연결된 것으로 보지 않는다.

### 입력
- 0과 1로 이루어진 2차원 정수 배열 `grid`
### 출력 
- 섬의 둘레 길이를 나타내는 정수
### 입출력 예시
```
예시 1:
입력:
grid = [[0,1,0],[1,1,1],[0,1,0]]
출력: 12

예시 2:
입력:
grid = [[1]]
출력: 4
```
- 예시 1: 십자가 모양의 섬이 만들어진다. 육지 칸은 총 5개이므로 섬의 바깥쪽 둘레는 12이다.
- 예시 2: 육지 칸이 1개이므로 섬의 둘레는 4이다.

### 문제 이해
- 입력: 0(물)과 1(육지)로 이루어진 2차원 정수 배열 `grid`
- 해야 할 일:
  - 모든 육지 칸을 찾는다.
  - 각 육지 칸의 상하좌우 네 방향을 확인한다.
  - 해당 방향이 격자 바깥이거나 물이면 둘레 길이를 1 증가시킨다. 
- 주의할 점:
  - 대각선 방향은 연결로 보지 않는다.
  - 격자 바깥은 모두 물로 취급한다.
  - 육지와 육지가 맞닿은 변은 섬 내부에 있으므로 둘레가 아니다.
- 반환 형식: 섬의 둘레 길이를 int형으로 반환한다.

### 접근법
1. Solution1: 육지 칸 기준 4방향 확인 방식 (권장)
   - 각 육지 칸마다 상하좌우를 확인하면서 물 또는 격자 바깥과 맞닿은 변만 둘레에 더한다.
2. Solution2: 전체 육지 둘레에서 공유 변 차감 방식
   - 육지 칸 하나를 4개의 변으로 보고, 인접한 육지끼리 공유하는 변을 2개씩 차감한다.


## Solution1: 육지 칸 기준 4방향 확인 방식 (권장)
### 코드 설계
1. grid의 행 개수와 열 개수를 구한다. 
2. 초기화할 변수
   - `int rows`: 전체 행 개수
   - `int cols`: 전체 열 개수
   - `int perimeter`: 섬의 둘레 길이를 누적할 변수 
   - `dr`: 상하좌우 방향 중 행 이동값 
   - `dc`: 상하좌우 방향 중 열 이동값 
3. 반복문 안에서 할 일
   - 모든 칸을 행 기준, 열 기준으로 순회한다.
   - 현재 칸이 물이면 둘레 계산 대상이 아니므로 건너뛴다.
   - 현재 칸이 육지이면 상하좌우 4방향을 확인한다.
   - 조건에 따른 갱신
     - 다음 위치가 격자 바깥이면 `perimeter++` 한다.
     - 다음 위치가 물이면 `perimeter++` 한다.
     - 다음 위치가 육지이면 둘레가 아니므로 아무것도 더하지 않는다.
   - 누적: 육지 칸이 물 또는 격자 바깥과 맞닿은 변의 개수만큼 `perimeter`에 누적한다.
   - 비교
     - `nextRow < 0`, `nextRow >= rows`, `nextCol < 0`, `nextCol >= cols`로 범위 초과 여부를 확인한다.
     - 범위 안이라면 `grid[nextRow][nextCol] == 0`인지 확인한다. 
4. 반환 형식: 계산된 섬의 둘레 길이 `perimeter`를 반환한다.

## 코드 구현
- 핵심 포인트 
- 시간복잡도: `O(R * C)`
  - R은 행 개수, C는 열 개수이다.
  - 모든 칸을 한 번씩 순회하고, 육지 칸마다 최대 4방향만 확인한다.
- 공간복잡도: `O(1)`


## Solution2: 전체 육지 둘레에서 공유 변 차감 방식
### 코드 설계
1. grid의 행 개수와 열 개수를 구한다. 
2. 초기화할 변수
   - `int rows`: 전체 행 개수
   - `int cols`: 전체 열 개수
   - `int perimeter`: 섬의 둘레 길이를 누적할 변수 
3. 반복문 안에서 할 일
   - 모든 칸을 순회한다.
   - 현재 칸이 물이면 건너뛴다.
   - 현재 칸이 육지이면 일단 둘레를 4 증가시킨다.
   - 조건에 따른 갱신
     - 위쪽 칸이 육지이면 공유 변이므로 2를 뺀다.
     - 왼쪽 칸이 육지이면 공유 변이므로 2를 뺀다.
   - 누적
     - 육지 칸 하나마다 기본적으로 4를 더한다.
     - 이미 확인한 위쪽, 왼쪽 육지와 맞닿은 경우만 중복 둘레를 제거한다.
   - 비교
     - `row > 0 && grid[row - 1][col] == 1`
     - `col > 0 && grid[row][col - 1] == 1`
4. 반환 형식: 계산된 섬의 둘레 길이 `perimeter`를 반환한다.

## 코드 구현
- 핵심 포인트 
- 시간복잡도: `O(R * C)`
  - 모든 칸을 한 번씩 순회한다.
- 공간복잡도: `O(1)`
