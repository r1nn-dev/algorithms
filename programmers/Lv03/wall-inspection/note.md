# Problem: Wall Inspection

### 문제
문제: 원형 외벽의 점검 지점에서 최소 인원으로 모든 지점을 점검하세요. 
```text
레스토랑의 구조는 완전히 동그란 모양이고 외벽의 총 둘레는 n미터이며, 외벽의 몇몇 지점은 추위가 심할 경우 손상될 수도 있는 취약한 지점들이 있습니다.
따라서 내부 공사 도중에도 외벽의 취약 지점들이 손상되지 않았는 지, 주기적으로 친구들을 보내서 점검을 하기로 했습니다.
다만, 빠른 공사 진행을 위해 점검 시간을 1시간으로 제한했습니다.
친구들이 1시간 동안 이동할 수 있는 거리는 제각각이기 때문에,
최소한의 친구들을 투입해 취약 지점을 점검하고 나머지 친구들은 내부 공사를 돕도록 하려고 합니다.
편의 상 레스토랑의 정북 방향 지점을 0으로 나타내며, 취약 지점의 위치는 정북 방향 지점으로부터 시계 방향으로 떨어진 거리로 나타냅니다.
또, 친구들은 출발 지점부터 시계, 혹은 반시계 방향으로 외벽을 따라서만 이동합니다.

외벽의 길이 n, 취약 지점의 위치가 담긴 배열 weak, 각 친구가 1시간 동안 이동할 수 있는 거리가 담긴 배열 dist가 매개변수로 주어질 때,
취약 지점을 점검하기 위해 보내야 하는 친구 수의 최소값을 return 하도록 solution 함수를 완성해주세요.
```
- 제한사항
  - n은 1 이상 200 이하인 자연수이다.
  - weak의 길이는 1 이상 15 이하이다.
    - 서로 다른 두 취약점의 위치가 같은 경우는 주어지지 않는다.
    - 취약 지점의 위치는 오름차순으로 정렬되어 주어진다.
    - weak의 원소는 0 이상 n - 1 이하인 정수이다.
  - dist의 길이는 1 이상 8 이하이다.
    - dist의 원소는 1 이상 100 이하인 자연수이다.
  - 친구들을 모두 투입해도 취약 지점을 전부 점검할 수 없는 경우에는 -1을 return 한.

### 입력
- 외벽의 길이 `n`
- 취약 지점의 위치가 담긴 배열 `weak` 
- 각 친구가 1시간 동안 이동할 수 있는 거리가 담긴 배열 `dist`
### 출력 
- 취약 지점을 점검하기 위해 보내야 하는 친구 수의 최솟값
### 입출력 예시
```
예시 1:
입력:
n = 12
weak = [1, 5, 6, 10]
dist = [2, 4, 6]
출력: 2 (최소 2명이 필요)

예시 2:
입력:
n = 12
weak = [1, 3, 4, 9, 10]
dist = [3, 5, 7]
출력: 1 (최소 1명이 필요)
```

### 문제 이해
- 입력:
  - 외벽의 길이 `n`
  - 취약 지점의 위치가 담긴 배열 `weak` 
  - 각 친구가 1시간 동안 이동할 수 있는 거리가 담긴 배열 `dist`
- 해야 할 일:
  - 모든 취약 지점을 점검하기 위해 필요한 친구 수의 최솟값을 구한다.
- 주의할 점:
  - 외벽은 원형이므로 마지막 지점과 처음 지점이 연결되어 있다.
  - 원형 배열을 그대로 처리하면 시작점과 끝점 처리가 복잡하다.
  - 따라서 weak 배열을 한 번 더 이어 붙여 원형 구조를 일자 구조처럼 만든다.
  - 친구는 시계 방향 또는 반시계 방향으로 이동할 수 있지만, 시작점을 모든 취약 지점으로 잡으면 한 방향만 고려해도 된다.
  - dist의 순서가 결과에 영향을 주므로 모든 순열을 확인해야 한다.
- 반환 형식:
  - 최소 친구 수를 int로 반환한다.
  - 불가능하면 -1을 반환한다.

### 접근법
1. Solution1: 원형 배열 펼치기 + 친구 순열 완전탐색
   - 원형 외벽을 일자 배열처럼 만들고, 모든 시작 지점과 모든 친구 투입 순서를 검사한다.
   - 제한사항에서 weak는 최대 15개, dist는 최대 8개이므로 완전탐색이 가능하다.


## Solution1: 원형 배열 펼치기 + 순열 완전탐색 방식 (권장)
### 코드 설계
1. 원형 구조를 일자 구조처럼 처리: 취약 지점 배열 weak을 2배 길이로 확장한다.
2. 친구들의 이동 가능 거리 dist로 만들 수 있는 모든 순열을 구한다.
3. 각 시작 취약 지점을 기준으로 점검을 시작한다: `weak[start]`
4. 반복문 안에서 할 일
   - 현재 친구가 점검할 수 있는 마지막 위치를 계산한다.
   - 다음 취약 지점이 그 범위 안에 있으면 같은 친구가 점검한다.
   - 범위를 벗어나면 다음 친구를 투입한다.
5. 사용한 친구 수의 최솟값을 갱신한다.
6. 반환 형식: 취약 지점을 점검하기 위해 보내야 하는 친구 수의 최소값을 int형으로 반환한다.
### 코드 구현
- 시간복잡도: `O(D! × W²)`
  - D: 친구 수
  - W: 취약 지점 수
  - 친구 수는 최대 8명이므로 8! = 40,320이다.
  - 취약 지점은 최대 15개이므로 완전탐색이 가능하다.
- 공간복잡도: `O(D! × D + W)`
  - 모든 순열을 저장하면 순열 개수만큼 추가 공간이 필요하다.
