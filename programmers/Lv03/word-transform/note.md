# Problem: Word Transformation

## 문제
문제: begin에서 target으로 변환하는데, words의 단어를 거쳐 한 글자씩만 변환 가능합니다. 최소 변환 횟수를 구하세요.
```
두 개의 단어 begin, target과 단어의 집합 words가 있습니다.
아래와 같은 규칙을 이용하여 begin에서 target으로 변환하는 가장 짧은 변환 과정을 찾으려고 합니다.
    1. 한 번에 한 개의 알파벳만 바꿀 수 있습니다.
    2. words에 있는 단어로만 변환할 수 있습니다.
두 개의 단어 begin, target과 단어의 집합 words가 매개변수로 주어질 때,
최소 몇 단계의 과정을 거쳐 begin을 target으로 변환할 수 있는지 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항:
  - 각 단어는 알파벳 소문자로만 이루어져 있다.
  - 각 단어의 길이는 3 이상 10 이하이며 모든 단어의 길이는 같다.
  - words에는 3개 이상 50개 이하의 단어가 있으며 중복되는 단어는 없다.
  - begin과 target은 같지 않다.
  - 변환할 수 없는 경우에는 0를 return 한다.

### 입력
- `begin`: 시작 단어
- `target`: 목표 단어
- `words`: 변환 과정에서 사용할 수 있는 단어 배열
### 출력
- begin에서 target으로 변환하는 최소 횟수
- 변환할 수 없으면 0
### 입출력 예시
```
예시 1:
입력:
begin = "hit", target: "cog"
words = ["hot", "dot", "dog", "lot", "log", "cog"]
경로: hit → hot → dot → dog → cog
출력:
return = 4

예시 2:
입력:
begin = "hit", target: "cog"
words = ["hot", "dot", "dog", "lot", "log"]
출력: 
return = 0
```

## Solution1: BFS - 변환 가능한 단어를 직접 탐색하는 방식 (권장)
### 코드 설계
1. `target`이 `words` 배열에 없으면 0을 반환한다.
2. `Queue<State>`를 생성해 시작 단어와 depth 0을 넣는다.
3. `visited` 배열로 이미 사용한 단어를 체크한다.
4. BFS를 수행하면서 현재 단어에서 한 글자만 다른 단어를 찾는다.
5. 변환 가능한 단어를 방문 처리하고 depth + 1로 큐에 넣는다.
6. `target`에 도달하면 depth를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(n^2 * m)`
    - `n`: 단어 개수
    - `m`: 단어 길이
  - 공간복잡도: `O(n)`

## Solution2: 그래프 구축 후 BFS 

### 코드 설계
1. `begin`이 `words` 배열에 없으면 0을 반환한다.
2. `target`의 인덱스를 찾는다.
3. 모든 단어 쌍을 비교해 한 글자만 다르면 양방향 간선으로 연결한다.
4. `begin`의 인덱스 0번부터 BFS를 시작한다.
5. BFS로 target 인덱스까지의 최단 거리를 구한다.
6. 도달하지 못하면 0을 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(n² × m)`
  - 그래프 구축에서 모든 단어 쌍을 비교한다.
- 공간복잡도: `O(n²)`
  - 최악의 경우 모든 단어가 서로 연결될 수 있다.
