# Problem: Farthest Node

### 문제 
문제: 1번 노드에서 모든 노드까지의 최단 거리를 구한 뒤, 가장 멀리 떨어진 노드의 개수를 반환하라.
```
n개의 노드가 있는 그래프가 있습니다. 각 노드는 1부터 n까지 번호가 적혀있습니다.
1번 노드에서 가장 멀리 떨어진 노드의 갯수를 구하려고 합니다.
가장 멀리 떨어진 노드란 최단경로로 이동했을 때 간선의 개수가 가장 많은 노드들을 의미합니다.

노드의 개수 n, 간선에 대한 정보가 담긴 2차원 배열 vertex가 매개변수로 주어질 때,
1번 노드로부터 가장 멀리 떨어진 노드가 몇 개인지를 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 노드의 개수 n은 2 이상 20,000 이하입니다.
  - 간선은 양방향이며 총 1개 이상 50,000개 이하의 간선이 있습니다.
  - vertex 배열 각 행 [a, b]는 a번 노드와 b번 노드 사이에 간선이 있다는 의미입니다.
  
### 입력
- 노드 개수 `n`
- 간선 정보 `edge`

### 출력
- 1번 노드에서 가장 멀리 떨어진 노드의 개수를 반환한다.

### 입출력 예시
```
예시1:
입력: 
n = 6
edge = [[3,6],[4,3],[3,2],[1,3],[1,2],[2,4],[5,2]]
출력: 3 
```
- 1번 노드에서 각 노드까지의 최단 거리를 구한 뒤, 가장 큰 거리값을 가진 노드의 개수를 센다.
- 1번 노드에서의 최단 거리:
  - 2번: 1
  - 3번: 1
  - 4번: 2
  - 5번: 2
  - 6번: 2
- 가장 먼 거리는 `2`이고,
- 그 거리를 가지는 노드는 `4, 5, 6`으로 총 `3개`이므로 정답은 `3`이다.

### 문제 이해
- 입력:
  - `n`: 노드 개수
  - `edge`: 양방향 간선 정보
- 해야 할 일:
  - 1번 노드에서 시작해 모든 노드까지의 최단 거리를 구한다.
  - 그중 가장 큰 거리값을 찾는다.
  - 가장 큰 거리값을 가진 노드의 개수를 반환한다.
- 주의할 점:
  - 간선은 양방향이다.
  - 간선 가중치가 없으므로 최단 거리는 지나간 간선의 개수이다.
  - 가중치가 없는 그래프의 최단 거리는 BFS로 구하는 것이 가장 적합하다.
  - 시작 노드인 1번의 거리는 0이다.
  - 방문하지 않은 노드와 시작 노드를 구분해야 하므로 거리 배열을 -1로 초기화하는 것이 좋다.
- 반환 형식: 가장 멀리 떨어진 노드의 개수를 int로 반환한다.

### 접근법 
1. Solution1: 인접 리스트 + BFS 방식
   - 그래프의 간선 가중치가 모두 동일하므로 BFS를 사용해 1번 노드부터 각 노드까지의 최단 거리를 구한다.
   - 이후 거리 배열을 순회하면서 가장 큰 거리값과 그 거리값을 가진 노드 개수를 계산한다.


## Solution1: 인접 리스트 + BFS 방식 
### 코드 설계
1. 1번부터 n번 노드까지 사용할 수 있도록 n + 1 크기의 인접 리스트를 만든다.
2. `edge` 배열을 순회하면서 양방향 간선을 간선을 저장한다.
    - `a -> b`
    - `b -> a`
3. 각 노드까지의 최단 거리를 저장할 `dist` 배열을 만든다.
    - 아직 방문하지 않은 노드는 -1로 초기화한다.
4. BFS 시작: 1번 노드를 큐에 넣고 `dist[1] = 0` 으로 설정한다.
5. 큐가 빌 때까지 반복한다.
   - 현재 노드를 꺼낸다.
   - 현재 노드와 연결된 다음 노드를 확인한다.
   - 아직 방문하지 않은 노드라면 `dist[next] = dist[current] + 1`로 갱신한다.
   - 다음 노드를 큐에 넣는다.
6. BFS가 끝나면 거리 `dist` 배열 전체를 순회하면서
    - 가장 큰 거리값 `maxDistance`를 찾는다.
    - `maxDistance`와 같은 값을 가진 노드 개수를 센다.
7. 가장 멀리 떨어진 노드의 개수를 반환한다.

## 코드 구현
- 핵심 포인트:
- 시간복잡도: `O(N + E)`
  - N: 노드 개수, E: 간선 개수
  - 인접 리스트 구성에 O(E)가 필요하다.
  - BFS에서 모든 노드와 간선을 최대 한 번씩 확인하므로 O(N + E)가 필요하다.
  - 거리 최댓값과 개수를 세는 데 O(N)이 필요하다.
- 공간복잡도: `O(N + E)`
  - 인접 리스트 + 거리 배열 + 큐
