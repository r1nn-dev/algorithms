# Problem: Farthest Node

### 문제 
문제: 1번 노드에서 모든 노드까지의 최단 거리를 구한 뒤, 가장 멀리 떨어진 노드의 개수를 반환하라.
```
n개의 노드가 있는 그래프가 있습니다. 각 노드는 1부터 n까지 번호가 적혀있습니다.
1번 노드에서 가장 멀리 떨어진 노드의 갯수를 구하려고 합니다.
가장 멀리 떨어진 노드란 최단경로로 이동했을 때 간선의 개수가 가장 많은 노드들을 의미합니다.

노드의 개수 n, 간선에 대한 정보가 담긴 2차원 배열 vertex가 매개변수로 주어질 때,
1번 노드로부터 가장 멀리 떨어진 노드가 몇 개인지를 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 노드의 개수 n은 2 이상 20,000 이하입니다.
  - 간선은 양방향이며 총 1개 이상 50,000개 이하의 간선이 있습니다.
  - vertex 배열 각 행 [a, b]는 a번 노드와 b번 노드 사이에 간선이 있다는 의미입니다.
  
### 입력
- 노드 개수 `n`
- 간선 정보 `edge`

### 출력
- 1번 노드에서 가장 멀리 떨어진 노드의 개수를 반환한다.

### 입출력 예시
```
예시1:
입력: 
n = 6
edge = [[3,6],[4,3],[3,2],[1,3],[1,2],[2,4],[5,2]]
출력: 3 
```
- 1번 노드에서 각 노드까지의 최단 거리를 구한 뒤, 가장 큰 거리값을 가진 노드의 개수를 센다.
- 1번 노드에서의 최단 거리:
  - 2번: 1
  - 3번: 1
  - 4번: 2
  - 5번: 2
  - 6번: 2
- 가장 먼 거리는 `2`이고,
- 그 거리를 가지는 노드는 `4, 5, 6`으로 총 `3개`이므로 정답은 `3`이다.

### 문제 이해
- 입력: 노드 개수 `n`과 양방향 간선 정보 `edge`
- 1번 노드에서 시작해 모든 노드까지의 최단 거리를 구한 뒤, 가장 먼 거리를 가진 노드의 개수를 세야 한다.
- 간선은 양방향이므로 `a -> b`, `b -> a`를 모두 저장해야 한다.
- 간선의 가중치가 없으므로 최단 거리는 지나간 간선의 개수이다.
- 아직 방문하지 않은 노드와 거리 0인 시작 노드를 구분해야 한다.
- 반환 형식: 가장 멀리 떨어진 노드의 개수를 `int`로 반환

### 접근법 — BFS(너비 우선 탐색)
- 가중치가 없는 격자에서 최단거리 구하는 문제는 BFS가 가장 정석이다.
    - 그래프 + 가중치 없음 → BFS
    - 격자도 그래프이므로 각 칸을 하나의 노드로 볼 수 있다.
    - 간선 가중치 모두 동일: 상하좌우 이동은 인접 노드 이동과 같다.
- BFS는 가까운 칸부터 레벨 순서대로 탐색하므로 어떤 칸에 처음 도달했을 때의 거리가 그 칸까지의 최단거리이다.
- 다익스트라는 가중치가 서로 다른 최단 거리 문제에서 사용한다.

### 코드 설계
1. 인접 리스트 `graph`를 만든다.
    - `List<List<Integer>> graph`
    - 1번부터 `n`번까지 사용할 수 있도록 `n + 1`개의 리스트 생성
2. `edge`를 순회하면서 양방향 간선을 인접 리스트에 추가한다. 
    - `a -> b`
    - `b -> a`
3. 각 노드까지의 최단 거리를 저장할 `dist[]` 배열을 만든다.
    - 아직 방문하지 않은 노드는 `-1`로 초기화한다.
4. BFS 시작: 1번 노드를 큐에 넣고 `dist[1] = 0` 으로 설정한다.
5. 큐가 빌 때까지 반복한다.
   - 큐에서 현재 노드를 하나 꺼낸다.
   - 현재 노드와 연결된 다음 노드를 확인한다.
   - 아직 방문하지 않았다면:
     - `dist[next] = dist[current] + 1`로 갱신
     - 다음 탐색을 위해 `next`를 큐에 넣는다.
6. BFS가 끝나면 거리 `dist` 배열 전체를 순회하면서
    - 가장 큰 거리값 `maxDist`를 찾고
    - 그 `maxDist`와 같은 값을 가진 노드의 개수를 센다.
7. 가장 멀리 떨어진 노드의 개수를 반환한다.


## 코드 구현

### Solution1: 인접 리스트 + BFS 방식 
- 핵심 포인트:
    - 그래프를 인접 리스트로 구성한다.
    - `dist[i] == -1`:  아직 방문하지 않은 노드
    - BFS를 통해 1번 노드부터 각 노드까지의 최단 거리를 구한다.
    - 마지막에 거리의 최댓값과 그 값을 가진 노드 개수를 계산한다.
- 시간복잡도: O(N + E)
  - 인접 리스트 구성: O(E)
  - BFS 탐: 각 노드/간선 최대 한 번씩 확인 → O(N + E)
  - 최댓값 및 개수 세기: O(N)
- 공간복잡도: `O(N + E)`
  - 인접 리스트 + 거리 배열 + 큐
