# Problem: Network

## 문제 
문제: 컴퓨터 n개가 있고 인접행렬 computers가 주어질 때, 네트워크의 개수를 반환하세요.
```
네트워크란 컴퓨터 상호 간에 정보를 교환할 수 있도록 연결된 형태를 의미합니다.
예를 들어, 컴퓨터 A와 컴퓨터 B가 직접적으로 연결되어있고,
컴퓨터 B와 컴퓨터 C가 직접적으로 연결되어 있을 때 컴퓨터 A와 컴퓨터 C도 간접적으로 연결되어 정보를 교환할 수 있습니다.
따라서 컴퓨터 A, B, C는 모두 같은 네트워크 상에 있다고 할 수 있습니다.
컴퓨터의 개수 n, 연결에 대한 정보가 담긴 2차원 배열 computers가 매개변수로 주어질 때, 네트워크의 개수를 return 하도록 solution 함수를 작성하시오.
```
- 제한사항
  - 컴퓨터의 개수 n은 1 이상 200 이하인 자연수입니다.
  - 각 컴퓨터는 0부터 n-1인 정수로 표현합니다.
  - i번 컴퓨터와 j번 컴퓨터가 연결되어 있으면 computers[i][j]를 1로 표현합니다.
  - computer[i][i]는 항상 1입니다.

### 입력
- `int n`: 컴퓨터의 개수
- `int[][] computers`: 컴퓨터 간 연결 정보를 나타내는 인접행렬
### 출력
- 네트워크의 개수를 int로 반환한다.
### 입출력 예시
```
예시 1:
입력:
n = 3
computers = [[1,1,0],[1,1,0],[0,0,1]]
출력:
result = 2 (네트워크 2개: {0,1}, {2})
```

## 풀이

### 문제 정보
- 플랫폼: 프로그래머스 
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/43162
- 난이도: Lv 00 (기준은 프로그래머스)
- 주제: 그래프, DFS/BFS, 연결 컴포넌트  
- 문제 핵심:
  - 인접행렬을 그래프로 해석
  - DFS/BFS로 연결된 컴포넌트 개수 세기
  - visited 배열로 중복 방지
- 최초 풀이: 260407 
- 수정 버전: 260702

### 문제 이해
- 입력:
  - `n`: 컴퓨터의 개수
  - `computers`: 컴퓨터 간 연결 정보를 나타내는 인접행렬
- 해야 할 일: 전체 컴퓨터를 순회하면서 연결된 컴퓨터 그룹의 개수를 센다.
- 주의할 점:
  - 직접 연결뿐만 아니라 간접 연결도 같은 네트워크로 본다.
  - `computers[i][j] == 1`이면 i번 컴퓨터와 j번 컴퓨터가 연결되어 있다는 의미이다.
  - `computers[i][i]`는 항상 1이지만, 자기 자신으로 이동하는 것은 네트워크 개수 계산에 큰 의미가 없다.
  - 이미 방문한 컴퓨터는 다시 탐색하지 않아야 한다.
- 반환 형식: 네트워크의 개수를 int로 반환한다.

### 접근법
1. Solution1: DFS 방식 (권장)
   - 아직 방문하지 않은 컴퓨터를 발견할 때마다 DFS로 연결된 컴퓨터 전체를 방문 처리한다.
   - 코드가 간결하고 연결 컴포넌트 개수 세기 문제에서 가장 직관적인 방식이다.
2. Solution2: BFS 방식
   - 아직 방문하지 않은 컴퓨터를 발견할 때마다 큐를 사용해 연결된 컴퓨터 전체를 방문 처리한다.
   - 재귀 호출을 사용하지 않기 때문에 스택 깊이 문제를 피할 수 있다.


## Solution1: 방식 (권장)

### 코드 설계
1. 방문 여부를 저장할 visited 배열을 만든다.
2. 전체 컴퓨터를 0번부터 n - 1번까지 순회한다.
3. 아직 방문하지 않은 컴퓨터를 발견하면 새로운 네트워크로 판단한다.
4. 해당 컴퓨터에서 DFS를 수행한다.
   - 현재 컴퓨터를 방문 처리한다.
   - 현재 컴퓨터와 연결된 다른 컴퓨터를 확인한다.
   - 연결되어 있고 아직 방문하지 않았다면 재귀적으로 DFS를 수행한다.
5. DFS가 한 번 시작될 때마다 네트워크 개수 count를 1 증가시킨다.
6. 모든 컴퓨터를 확인한 뒤 count를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N²)`
  - 각 노드마다 인접행렬의 한 행을 확인한다.
- 공간복잡도: `O(N)`
  - 방문 배열과 재귀 호출 스택에 O(N)이 필요하다.


## Solution2: 방식

### 코드 설계
1. 방문 여부를 저장할 visited 배열을 만든다.
2. 전체 컴퓨터를 0번부터 n - 1번까지 순회한다.
3. 아직 방문하지 않은 컴퓨터를 발견하면 새로운 네트워크로 판단한다.
4. 해당 컴퓨터를 큐에 넣고 BFS를 시작한다.
   - 현재 컴퓨터를 꺼낸다.
   - 현재 컴퓨터와 연결된 다른 컴퓨터를 확인한다.
   - 연결되어 있고 아직 방문하지 않았다면 방문 처리 후 큐에 넣는다.
5. BFS가 한 번 시작될 때마다 네트워크 개수 count를 1 증가시킨다.
6. 모든 컴퓨터를 확인한 뒤 count를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N²)`
  - 인접행렬을 사용하므로 각 컴퓨터마다 연결 여부를 확인한다.
- 공간복잡도: `O(N)`
  - 방문 배열과 큐에 O(N)이 필요하다.
