# Problem: Immigration Inspection

### 문제
문제: 심사관별 심사 시간이 주어질 때, n명을 모두 심사하는 최소 시간을 구하세요.
```text
n명이 입국심사를 위해 줄을 서서 기다리고 있습니다. 각 입국심사대에 있는 심사관마다 심사하는데 걸리는 시간은 다릅니다.
처음에 모든 심사대는 비어있습니다. 한 심사대에서는 동시에 한 명만 심사를 할 수 있습니다.
가장 앞에 서 있는 사람은 비어 있는 심사대로 가서 심사를 받을 수 있습니다.
하지만 더 빨리 끝나는 심사대가 있으면 기다렸다가 그곳으로 가서 심사를 받을 수도 있습니다.
모든 사람이 심사를 받는데 걸리는 시간을 최소로 하고 싶습니다.

입국심사를 기다리는 사람 수 n, 각 심사관이 한 명을 심사하는데 걸리는 시간이 담긴 배열 times가 매개변수로 주어질 때,
모든 사람이 심사를 받는데 걸리는 시간의 최솟값을 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 입국심사를 기다리는 사람은 1명 이상 1,000,000,000명 이하입니다.
  - 각 심사관이 한 명을 심사하는데 걸리는 시간은 1분 이상 1,000,000,000분 이하입니다.
  - 심사관은 1명 이상 100,000명 이하입니다.

### 입력
- `n`: 입국심사를 기다리는 사람 수
- `times`: 각 심사관이 한 명을 심사하는 데 걸리는 시간 배열
### 출력 
- 모든 사람이 입국심사를 받는 데 걸리는 최소 시간을 long 타입으로 반환한다. 
### 입출력 예시
```
예시 1:
입력:
n = 6
times = [7, 10]
출력: 28 (7분 심사관 4명, 10분 심사관 2명 이용)
```

  - 주어진 시간에 몇 명을 심사할 수 있는지 계산

### 문제 이해
- 입력:
  - `n`: 입국심사를 기다리는 사람 수
  - `times`: 각 심사관이 한 명을 심사하는 데 걸리는 시간 배열
- 해야 할 일: 모든 사람을 심사할 수 있는 최소 시간을 구해야 한다.
- 주의할 점:
  - 사람을 한 명씩 심사대에 배정하는 방식으로 풀면 안 된다.
  - n은 최대 1,000,000,000명이고, 심사관 수도 최대 100,000명이므로 직접 시뮬레이션하면 시간 초과가 발생한다.
  - 가능한 시간의 범위가 매우 크기 때문에 int가 아니라 long을 사용해야 한다.
  - 어떤 시간 `mid`가 주어졌을 때, 각 심사관이 처리할 수 있는 사람 수는 `mid / 심사시간`이다.
- 반환 형식: 모든 사람이 심사를 받는 데 필요한 최소 시간을 long으로 반환한다.

### 접근법
1. Solution1: 이진탐색 기반 Parametric Search (권장)
   - 정답이 될 수 있는 시간을 범위로 잡고, 해당 시간 안에 n명을 심사할 수 있는지 판단하면서 최소 시간을 찾는다.

### 코드 설계
1. 가장 빠른 심사 시간을 찾는다.
   - 최소 심사 시간을 `minTime`에 저장한다.
2. 이진탐색 범위를 설정한다.
   - `left = 1`
   - `right = minTime * n`
   - 가장 빠른 심사관 혼자 모든 사람을 심사하는 시간이 최악의 답이 될 수 있다.
3. `answer`를 `right`로 초기화한다.
   - 최소 시간을 찾는 과정에서 가능한 시간이 나오면 계속 갱신한다.
4. 이진탐색을 수행한다.
   - `mid = left + (right - left) / 2`
   - mid분 동안 모든 심사관이 처리할 수 있는 총 인원을 계산한다.
5. 처리 가능 인원이 n 이상인지 확인한다.
   - `count >= n`: mid분 안에 모두 심사 가능하다.
     - `answer = mid`
     - 더 작은 시간이 가능한지 확인하기 위해 `right = mid - 1`
   - `count < n`: 시간이 부족하다.
     - 더 큰 시간이 필요하므로 `left = mid + 1`
6. 최종적으로 `answer`를 반환한다.

## 코드 구현

### Solution1: 이진탐색 기반 Parametric Search (권장)
- 핵심 포인트
  - 정답이 될 수 있는 시간을 이진탐색한다.
  - 특정 시간 안에 심사 가능한 총 인원 수를 계산한다.
  - 가능한 시간 중 가장 작은 값을 찾는다.
  - 시간 범위가 매우 크므로 long을 사용한다.
  - count가 n 이상이 되면 더 계산하지 않고 반복문을 종료해 오버플로우를 방지한다.
- 시간복잡도: `O(M log T)`
  - 심사관 수를 M, 정답 시간 범위를 T라고 하면
  - 이진탐색은 O(log T)번 수행된다.
  - 각 탐색마다 심사관 배열을 한 번 순회하므로 O(M)이 걸린다.
- 공간복잡도: `O(1)`
  - 추가 배열을 사용하지 않는다.
