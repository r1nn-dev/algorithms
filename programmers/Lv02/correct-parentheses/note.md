# Problem: Correct Parentheses

### 문제
문제: 괄호의 짝이 올바르게 맞으면 true, 아니면 false를 반환하세요.
```text
괄호가 바르게 짝지어졌다는 것은 '(' 문자로 열렸으면 반드시 짝지어서 ')' 문자로 닫혀야 한다는 뜻입니다.
예를 들어
- "()()" 또는 "(())()" 는 올바른 괄호입니다.
- ")()(" 또는 "(()(" 는 올바르지 않은 괄호입니다.
'(' 또는 ')' 로만 이루어진 문자열 s가 주어졌을 때, 문자열 s가 올바른 괄호이면 true를 return 하고, 
올바르지 않은 괄호이면 false를 return 하는 solution 함수를 완성해 주세요.
```
- 제한사항
  - 문자열 s의 길이 : 100,000 이하의 자연수
  - 문자열 s는 '(' 또는 ')' 로만 이루어져 있다.

### 입력
- '(' 또는 ')' 로만 이루어진 문자열 s
### 출력 
- 문자열 s가 올바른 괄호이면 true
- 올바르지 않은 괄호이면 false
### 입출력 예시
```
예시 1:
입력: s = "()()"
출력: true

예시 2:
입력: s = ")()())"
출력: false
```

  - 열린 괄호는 증가, 닫힌 괄호는 감소
  - 언제든 음수가 되면 false

### 문제 이해
- 입력: 괄호 문자 '(', ')' 로만 이루어진 문자열 s
- 해야 할 일:
  - 문자열을 왼쪽부터 오른쪽으로 확인하면서 괄호의 짝이 올바른지 판단한다.
- 주의할 점:
  - 닫는 괄호 ')' 가 먼저 나오면 올바른 괄호가 아니다.
  - 중간 어느 시점이든 닫힌 괄호 개수가 열린 괄호 개수보다 많아지면 false 이다.
  - 문자열을 끝까지 확인했을 때 열린 괄호가 남아 있어도 false 이다.
  - 문자열 길이가 최대 100,000이므로 중첩 반복문 없이 한 번만 순회해야 한다.
- 반환 형식: 올바른 괄호이면 true, 올바르지 않은 괄호이면 false

### 접근법
1. Solution1: 카운터 방식 (권장)
   - Stack의 원리만 이용 
   - 열린 괄호 개수만 count 변수로 관리한다.
   - 괄호 종류가 '(', ')' 한 쌍뿐이므로 실제 Stack에 문자를 저장할 필요 없다.
2. Solution2: Stack 방식
   - `Stack<Character>`, `Deque<Character>`
   - '(' 를 만나면 push, ')' 를 만나면 pop
   - 괄호 종류가 '(', ')' 한 쌍뿐이므로 실제 Stack에 문자를 저장하는 것은 공간 낭비이다.

### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 열린 괄호 개수를 저장할 변수 count 를 0으로 초기화한다.
2. 초기화할 변수
   - `int count = 0;` 현재까지 닫히지 않은 열린 괄호의 개수
3. 반복문 안에서 할 일
   - 문자열 s를 왼쪽부터 한 글자씩 확인한다.
   - 현재 문자가 '(' 이면 count 를 증가, ')' 이면 count 를 감소시킨다.
   - 감소한 뒤 `count < 0` 이 되면 닫을 수 있는 열린 괄호가 없다는 뜻이므로 즉시 false 를 반환한다.
4. 반환 형식
   - 반복문이 끝난 뒤 count == 0 이면 모든 괄호가 정확히 닫힌 것이므로 true
   - count != 0 이면 열린 괄호가 남아 있는 것이므로 false

## 코드 구현

### Solution1: 카운터 방식 (권장)
- 핵심 포인트
  - 실제 Stack을 쓰지 않고 열린 괄호 개수만 관리한다.
  - '(' 는 아직 닫히지 않은 괄호가 하나 생긴 것이므로 `count++`
  - ')' 는 열린 괄호 하나를 닫는 것이므로 `count--` 
- 시간복잡도: O(N)
- 공간복잡도: O(1)

### Solution2: Stack 방식
- 핵심 포인트
  - 실제 Stack 사용: Stack에 넣고 꺼낸다. 
- 시간복잡도: O(N)
- 공간복잡도: O(N)
