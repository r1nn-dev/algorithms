# Problem: Correct Parentheses

### 문제
문제: 괄호의 짝이 올바르게 맞으면 true, 아니면 false를 반환하세요.
```text
괄호가 바르게 짝지어졌다는 것은 '(' 문자로 열렸으면 반드시 짝지어서 ')' 문자로 닫혀야 한다는 뜻입니다.
예를 들어
  - "()()" 또는 "(())()" 는 올바른 괄호입니다.
  - ")()(" 또는 "(()(" 는 올바르지 않은 괄호입니다.
'(' 또는 ')' 로만 이루어진 문자열 s가 주어졌을 때, 문자열 s가 올바른 괄호이면 true를 return 하고, 
올바르지 않은 괄호이면 false를 return 하는 solution 함수를 완성해 주세요.
```
- 제한사항
  - 문자열 s의 길이 : 100,000 이하의 자연수
  - 문자열 s는 `'('` 또는 `')'` 로만 이루어져 있다.

### 입력
- `'('` 또는 `')'` 로만 이루어진 문자열 `s`
### 출력 
- 문자열 `s`가 올바른 괄호이면 `true`
- 올바르지 않은 괄호이면 `false`
### 입출력 예시
```
예시 1:
입력: s = "()()"
출력: true

예시 2:
입력: s = ")()())"
출력: false
```

  - 언제든 음수가 되면 false

### 문제 이해
- 입력: 괄호 문자 `'('`, `')'` 로만 이루어진 문자열 `s`
- 해야 할 일:
  - 문자열을 왼쪽부터 오른쪽으로 확인하면서 괄호의 짝이 올바른지 판단한다.
- 주의할 점:
  - `')'`가 먼저 나오면 닫을 수 있는 `'('`가 없으므로 올바른 괄호가 아니다.
  - 순회 도중 어느 시점이든 닫힌 괄호 개수가 열린 괄호 개수보다 많아지면 `false`이다.
  - 문자열 끝까지 확인했을 때 열린 괄호가 남아 있으면 `false`이다.
  - 문자열 길이가 최대 100,000이므로 중첩 반복문 없이 한 번만 순회해야 한다.
- 반환 형식: 올바른 괄호이면 `true`, 올바르지 않은 괄호이면 `false`

### 접근법
1. Solution1: 카운터 방식 (권장)
   - Stack의 원리만 이용 
   - 괄호 종류가 `'('`, `')'` 한 쌍뿐이므로 실제 Stack을 사용하지 않고 열린 괄호의 개수만 관리한다.
   - 공간을 거의 사용하지 않으면서 빠르게 해결할 수 있다.
2. Solution2: Stack 방식
   - 실제 Stack 자료구조를 사용하여 괄호의 짝을 직접 관리한다.
   - `'('`가 나오면 Stack에 push하고, `')'`가 나오면 pop하여 짝을 맞춘다.
   - Stack이 비어 있는데 `')'`가 나오면 올바르지 않은 괄호이다.


## Solution1: 카운터 방식 (권장)
### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 문자열 길이가 홀수인지 확인한다.
   - 괄호는 반드시 두 개가 한 쌍이므로 길이가 홀수이면 올바른 괄호가 될 수 없다.
2. 초기화할 변수
   - `int count = 0;`: 아직 닫히지 않은 `'('`의 개수를 저장한다.
3. 반복문 안에서 할 일
   - `s.charAt(i)`로 현재 문자를 확인한다.
   - `'('`이면 `count++`, `')'`이면 `count--`
   - `count < 0`이면 닫을 수 있는 열린 괄호가 없다는 뜻이므로 `false`를 반환한다.
4. 반환 형식: 반복문 종료 후 `count == 0`을 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N)`
- 공간복잡도: `O(1)`


## Solution2: Stack 방식
### 코드 설계
1. 문자열 길이가 홀수이면 바로 `false`를 반환한다.
2. 초기화할 변수
   - `Stack<Character> stack = new Stack<>();`
3. 반복문 안에서 할 일
   - `'('`이면 stack에 push
   - `')'`이면 stack에서 pop
   - pop하려는데 stack이 비어 있으면 `false`
4. 반환 형식: 반복문 종료 후 stack이 비어 있으면 `true`, 아니면 `false`

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N)`
- 공간복잡도: `O(N)`
