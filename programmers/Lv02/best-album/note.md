# Problem: Best Album

### 문제
문제: 장르별 재생 횟수가 높은 순서로, 각 장르 내에서는 재생 횟수가 높은 노래 2개를 반환하세요.
```text
스트리밍 사이트에서 장르 별로 가장 많이 재생된 노래를 두 개씩 모아 베스트 앨범을 출시하려 합니다.
노래는 고유 번호로 구분하며, 노래를 수록하는 기준은 다음과 같습니다.
  1. 속한 노래가 많이 재생된 장르를 먼저 수록합니다.
  2. 장르 내에서 많이 재생된 노래를 먼저 수록합니다.
  3. 장르 내에서 재생 횟수가 같은 노래 중에서는 고유 번호가 낮은 노래를 먼저 수록합니다.
노래의 장르를 나타내는 문자열 배열 genres와 노래별 재생 횟수를 나타내는 정수 배열 plays가 주어질 때,
베스트 앨범에 들어갈 노래의 고유 번호를 순서대로 return 하도록 solution 함수를 완성하세요.
```
- 제한사항
  - genres[i]는 고유번호가 i인 노래의 장르이다.
  - plays[i]는 고유번호가 i인 노래가 재생된 횟수이다.
  - genres와 plays의 길이는 같으며, 이는 1 이상 10,000 이하이다.
  - 장르 종류는 100개 미만이다.
  - 장르에 속한 곡이 하나라면, 하나의 곡만 선택한다.
  - 모든 장르는 재생된 횟수가 다르다.

### 입력
- `String[] genres`
- `int[] plays`
### 출력 
- 베스트 앨범에 수록할 노래의 고유 번호 배열 int[]
### 입출력 예시
```
예시 1:
입력:
genres = ["classic", "pop", "classic", "classic", "pop"]
plays = [500, 600, 150, 800, 2500]
출력: [4, 1, 3, 0]
```


## 풀이

### 문제 정보
- 플랫폼: 프로그래머스 
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42579
- 난이도: Lv 02 
- 주제: 해시맵, 정렬, 우선순위
- 문제 핵심:
  - HashMap으로 장르별 정보 관리
  - 2중 정렬 (장르순, 곡순)
- 최초 풀이: 260616 
- 수정 버전: 260622

### 문제 이해
- 입력:
  - `genres[i]`: 고유 번호 i번 노래의 장르
  - `plays[i]`: 고유 번호 i번 노래의 재생 횟수
- 해야 할 일:
  - 장르별 총 재생 횟수를 계산한다.
  - 총 재생 횟수가 높은 장르부터 처리한다.
  - 각 장르에서 재생 횟수가 높은 노래를 최대 2개 선택한다.
- 주의할 점:
  - 장르 정렬 기준과 곡 정렬 기준이 다르다.
  - 장르는 해당 장르에 속한 모든 곡의 총 재생 횟수 기준으로 정렬한다.
  - 곡은 개별 재생 횟수 기준으로 정렬한다.
  - 같은 장르 안에서 재생 횟수가 같으면 고유 번호가 낮은 곡을 먼저 선택한다.
  - 장르에 속한 곡이 하나뿐이면 1개만 선택한다.
  - 모든 장르의 총 재생 횟수는 서로 다르므로 장르 간 동률 처리는 필요 없다.
- 반환 형식: 선택된 노래의 고유 번호를 순서대로 담은 int[] 배열을 반환한다.

### 접근법
1. Solution1: HashMap + 정렬 방식 
   - 장르별 총 재생 횟수와 장르별 곡 목록을 HashMap으로 관리한 뒤, 문제의 정렬 기준에 맞게 장르와 곡을 각각 정렬한다.

### 코드 설계
1. 실제 코드의 첫 번째 동작
   - 각 노래를 순회하면서 장르별 총 재생 횟수를 누적한다.
   - 동시에 장르별 노래 목록에 해당 노래 정보를 저장한다.
2. 초기화할 변수
   - `genreTotalMap`
     - key: 장르명
     - value: 해당 장르의 총 재생 횟수
   - `genreSongsMap`
     - key: 장르명
     - value: 해당 장르에 속한 노래 목록
   - `answerList`: 최종적으로 선택된 노래의 고유 번호를 저장하는 리스트
3. 반복문 안에서 할 일
   - `genres`와 `plays`를 순회한다.
   - 현재 노래의 장르와 재생 횟수를 가져온다.
   - `genreTotalMap`에 장르별 총 재생 횟수를 누적한다.
   - `genreSongsMap`에 현재 노래의 고유 번호와 재생 횟수를 저장한다.
4. 반환 형식
   - `answerList`에 저장된 Integer 값을 int[] 배열로 변환하여 반환한다.

## 코드 구현

### Solution1: HashMap + 정렬 방식 
- 핵심 포인트
  - 장르 정렬과 곡 정렬을 분리해서 생각한다.
  - 장르 정렬 기준은 장르별 총 재생 횟수이다.
  - 곡 정렬 기준은 개별 재생 횟수와 고유 번호이다.
  - 각 장르에서 최대 2곡까지만 선택한다.
- 시간복잡도: `O(N log N)`
  - 전체 곡 수를 N, 장르 수를 G라고 할 때 O(N + G log G + ΣKᵢ log Kᵢ)이다.
  - Kᵢ: 각 장르에 속한 곡의 개수
- 공간복잡도: `O(N)`
  - 장르별 총 재생 횟수를 저장하는 HashMap에 O(G)가 필요하다.
  - 장르별 곡 목록을 저장하는 HashMap에 O(N)가 필요하다.
  - 정답 리스트에 최대 O(N)가 필요하다.
