# Problem: 구명보트

주제: 그리디, 정렬, 투 포인터  

문제: 사람들의 몸무게와 보트 무게 제한이 주어질 때, 모두를 구출하는데 필요한 최소 보트 수를 구하세요.
```
무인도에 갇힌 사람들을 구명보트를 이용하여 구출하려고 합니다. 구명보트는 작아서 한 번에 최대 2명씩 밖에 탈 수 없고, 무게 제한도 있습니다.
예를 들어, 사람들의 몸무게가 [70kg, 50kg, 80kg, 50kg]이고 구명보트의 무게 제한이 100kg이라면
2번째 사람과 4번째 사람은 같이 탈 수 있지만 1번째 사람과 3번째 사람의 무게의 합은 150kg이므로 구명보트의 무게 제한을 초과하여 같이 탈 수 없습니다.
구명보트를 최대한 적게 사용하여 모든 사람을 구출하려고 합니다.
사람들의 몸무게를 담은 배열 people과 구명보트의 무게 제한 limit가 매개변수로 주어질 때,
모든 사람을 구출하기 위해 필요한 구명보트 개수의 최솟값을 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 무인도에 갇힌 사람은 1명 이상 50,000명 이하이다.
  - 각 사람의 몸무게는 40kg 이상 240kg 이하이다.
  - 구명보트의 무게 제한은 40kg 이상 240kg 이하이다.
  - 구명보트의 무게 제한은 항상 사람들의 몸무게 중 최댓값보다 크게 주어지므로 사람들을 구출할 수 없는 경우는 없다.
  - 구명보트는 한 번에 최대 2명까지만 탈 수 있다.


### 입력
- `int[] people`: 사람들의 몸무게 배열
- `int limit`: 구명보트의 무게 제한
### 출력
- 모든 사람을 구출하기 위해 필요한 구명보트 개수의 최솟값을 정수로 반환한다.
### 입출력 예시
```
예시 1:
입력:
people = [70, 50, 80, 50]
limit = 100
출력: 3 (70+50, 80, 50)
```

## 풀이
### 문제 정보
- 플랫폼: 프로그래머스

- 문제 핵심:
- 가장 무거운 사람과 가장 가벼운 사람을 짝지어보기
  - 투 포인터로 효율적으로 처리
- 최초 풀이: 260629

### 문제 이해
- 입력:
  - `people`: 사람들의 몸무게가 담긴 배열
  - `limit`: 구명보트의 무게 제한
- 해야 할 일:
  - 모든 사람을 구출해야 한다.
  - 한 보트에는 최대 2명까지만 탈 수 있다.
  - 각 보트에 탄 사람들의 몸무게 합은 `limit` 이하여야 한다.
  - 필요한 구명보트 개수의 최솟값을 구해야 한다.
- 주의할 점:
  - 보트에는 최대 2명까지만 탈 수 있으므로 3명 이상을 묶는 경우는 고려하지 않는다.
  - 가장 무거운 사람은 반드시 현재 보트 하나를 사용해야 한다.
  - 가장 무거운 사람이 가장 가벼운 사람과도 같이 탈 수 없다면, 다른 누구와도 같이 탈 수 없다.
  - 따라서 가장 무거운 사람을 기준으로 가장 가벼운 사람과 짝지어보는 전략이 유효하다.
- 반환 형식: 필요한 구명보트 개수의 최솟값을 int로 반환한다.

### 접근법
1. Solution1: 정렬 + 투 포인터 방식 (권장)
   - 사람들의 몸무게를 오름차순으로 정렬한다.
   - 가장 가벼운 사람을 가리키는 `left`, 가장 무거운 사람을 가리키는 `right` 포인터를 사용한다.
   - 가장 무거운 사람과 가장 가벼운 사람을 함께 태울 수 있으면 둘 다 처리한다.
   - 함께 태울 수 없으면 가장 무거운 사람만 혼자 태운다.
   - 매번 보트 하나를 사용하므로 `answer`를 1 증가시킨다.
2. Solution2: 카운팅 배열 + 투 포인터 방식
   - 사람의 몸무게가 40kg 이상 240kg 이하로 범위가 작다는 제한사항을 활용한다.
   - 각 몸무게별 사람 수를 `count` 배열에 저장한다.
   - 가장 가벼운 몸무게와 가장 무거운 몸무게를 포인터처럼 이동시키며 처리한다.


## Solution1: 정렬 + 투 포인터 방식 (권장)

### 코드 설계
1. `Arrays.sort(people)`로 몸무게를 오름차순 정렬한다.
2. 가장 가벼운 사람의 위치를 나타내는 `left`를 0으로 초기화한다.
3. 가장 무거운 사람의 위치를 나타내는 `right`를 `people.length - 1`로 초기화한다.
4. 필요한 보트 수를 저장할 `answer`를 0으로 초기화한다.
5. `left <= right`인 동안 반복한다.
   - 가장 가벼운 사람과 가장 무거운 사람이 함께 탈 수 있는지 확인한다.
   - 함께 탈 수 있으면 두 사람을 같은 보트에 태우고 `left++`, `right--`를 수행한다.
   - 함께 탈 수 없으면 가장 무거운 사람만 혼자 태우고 `right--`를 수행한다.
   - 보트 하나를 사용했으므로 `answer++`를 수행한다.
6. 반복이 끝나면 `answer`를 반환한다.

### 코드 구현
- 핵심 포인트 
- 시간복잡도: `O(N log N)`
  - 정렬에 O(N log N)이 걸린다.
  - 투 포인터 탐색은 각 사람을 한 번씩 처리하므로 O(N)이다.
- 공간복잡도: `O(1)`
  - 입력 배열을 직접 정렬하고, 추가로 큰 자료구조를 사용하지 않는다.
  - 단, Java의 정렬 내부 구현에 따른 추가 공간은 제외한다.
