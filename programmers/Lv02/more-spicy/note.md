# Problem: More Spicy

### 문제
문제: 음식들의 스코빌 지수가 주어질 때, 모든 음식의 스코빌 지수를 K 이상으로 만드는 최소 섞기 횟수를 구하세요.
```text
매운 것을 좋아하는 Leo는 모든 음식의 스코빌 지수를 K 이상으로 만들고 싶습니다.
모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 Leo는 스코빌 지수가 가장 낮은 두 개의 음식을
아래와 같이 특별한 방법으로 섞어 새로운 음식을 만듭니다.

섞은 음식의 스코빌 지수 = 가장 맵지 않은 음식의 스코빌 지수 + (두 번째로 맵지 않은 음식의 스코빌 지수 * 2)

Leo는 모든 음식의 스코빌 지수가 K 이상이 될 때까지 반복하여 섞습니다.
Leo가 가진 음식의 스코빌 지수를 담은 배열 scoville과 원하는 스코빌 지수 K가 주어질 때,
모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 섞어야 하는 최소 횟수를 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - `scoville`의 길이는 2 이상 1,000,000 이하이다.
  - `K`는 0 이상 1,000,000,000 이하이다.
  - `scoville`의 원소는 각각 0 이상 1,000,000 이하이다.
  - 모든 음식의 스코빌 지수를 `K` 이상으로 만들 수 없는 경우에는 `-1`을 return 한다.

### 입력
- `int[] scoville`: 음식들의 스코빌 지수 배열
- `int K`: 목표 스코빌 지수
### 출력 
- 모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 필요한 최소 섞기 횟수를 반환한다.
- 만들 수 없는 경우 -1을 반환한다.
### 입출력 예시
```
예시 1:
입력:
scoville = [1, 2, 3, 9, 10, 12],
K = 7
출력: 2
```

### 문제 이해
- 입력:
  - `scoville`: 음식들의 스코빌 지수가 담긴 배열
  - `K`: 목표 스코빌 지수
- 해야 할 일:
  - 모든 음식의 스코빌 지수가 `K` 이상이 되도록 음식을 섞어야 한다.
  - 한 번 섞을 때마다 가장 맵지 않은 음식 2개를 꺼낸다.
  - 두 음식을 섞어 새 스코빌 지수를 만든 뒤 다시 음식 목록에 넣는다.
  - 모든 음식이 `K` 이상이 되는 최소 섞기 횟수를 구한다.
- 주의할 점:
  - 매번 가장 작은 스코빌 지수 2개를 찾아야 한다.
  - 단순 정렬을 반복하면 `scoville.length`가 최대 1,000,000이므로 효율성 문제가 발생한다.
  - 음식이 1개만 남았는데 그 음식의 스코빌 지수가 K보다 작다면 더 이상 섞을 수 없으므로 `-1`을 반환해야 한다.
  - K가 0일 수 있으므로 처음부터 모든 음식이 조건을 만족하면 0을 반환해야 한다.
  - 새로 만들어지는 스코빌 지수는 int 범위를 넘을 수 있으므로 계산과 저장은 long으로 처리하는 것이 안전하다.
- 반환 형식:
  - 필요한 최소 섞기 횟수를 int로 반환한다.
  - 만들 수 없는 경우 -1을 반환한다.

### 접근법
1. Solution1: PriorityQueue 방식 (권장) 
   - 가장 작은 값 2개를 반복해서 꺼내야 하므로 Min Heap 구조인 `PriorityQueue`를 사용한다.
   - Java의 `PriorityQueue`는 기본적으로 오름차순 우선순위 큐이므로 가장 작은 값이 먼저 나온다.
   - 현재 가장 작은 값이 `K` 이상이면 모든 음식이 조건을 만족한 것이다.
   - 현재 가장 작은 값이 `K`보다 작으면 가장 작은 음식 2개를 꺼내 섞고, 새 스코빌 지수를 다시 넣는다.


## Solution1: 방식 (권장)
### 코드 설계
1. `PriorityQueue<Long>` 생성한다.
2. `scoville` 배열의 모든 값을 `PriorityQueue`에 넣는다.
3. 섞기 횟수를 저장할 `count`를 0으로 초기화한다.
4. `pq.peek()`가 `K`보다 작은 동안 반복한다.
   - `pq.size()`가 2보다 작으면 더 이상 섞을 수 없으므로 `-1`을 반환한다.
   - 가장 맵지 않은 음식 `first`를 꺼낸다.
   - 두 번째로 맵지 않은 음식 `second`를 꺼낸다.
   - 새 스코빌 지수 `mixed = first + second * 2`를 계산한다.
   - 새 음식을 다시 `pq`에 넣는다.
   - 섞기 횟수 `count`를 1 증가시킨다.
5. 반환 형식: 모든 음식의 스코빌 지수가 `K` 이상이면 `count`를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N log N)`
  - 초기 N개의 값을 우선순위 큐에 넣는 과정에서 O(N log N)이 걸릴 수 있다.
  - 섞는 과정에서 `poll()` 2번, `offer()` 1번이 반복된다.
  - 각 연산은 Heap 높이에 따라 O(log N)이다.
- 공간복잡도: `O(N)`
  - 모든 스코빌 지수를 우선순위 큐에 저장한다.
