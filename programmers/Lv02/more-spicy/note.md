# Problem: More Spicy

### 문제
문제: 음식들의 스코빌 지수가 주어질 때, 모든 음식의 스코빌 지수를 K 이상으로 만드는 최소 섞기 횟수를 구하세요.
```text
매운 것을 좋아하는 Leo는 모든 음식의 스코빌 지수를 K 이상으로 만들고 싶습니다.
모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 Leo는 스코빌 지수가 가장 낮은 두 개의 음식을
아래와 같이 특별한 방법으로 섞어 새로운 음식을 만듭니다.

섞은 음식의 스코빌 지수 = 가장 맵지 않은 음식의 스코빌 지수 + (두 번째로 맵지 않은 음식의 스코빌 지수 * 2)

Leo는 모든 음식의 스코빌 지수가 K 이상이 될 때까지 반복하여 섞습니다.
Leo가 가진 음식의 스코빌 지수를 담은 배열 scoville과 원하는 스코빌 지수 K가 주어질 때,
모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 섞어야 하는 최소 횟수를 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - scoville의 길이는 2 이상 1,000,000 이하이다.
  - K는 0 이상 1,000,000,000 이하이다.
  - scoville의 원소는 각각 0 이상 1,000,000 이하이다.
  - 모든 음식의 스코빌 지수를 K 이상으로 만들 수 없는 경우에는 -1을 return 한다.

### 입력
- 정수 배열 `scoville`
- 목표 스코빌 지수 `K`
### 출력 
- 모든 음식의 스코빌 지수를 `K` 이상으로 만들기 위해 필요한 최소 섞기 횟수
- 만들 수 없는 경우 `-1`
### 입출력 예시
```
예시 1:
입력:
scoville = [1, 2, 3, 9, 10, 12],
K = 7
출력: 2
```

- 문제 핵심:
  - Min Heap으로 가장 맵지 않은 음식부터 처리
  - PriorityQueue 적극 활용
- 최초 풀이: 260516

### 문제 이해
- 입력:
  - 음식들의 스코빌 지수를 담은 정수 배열 `scoville`
  - 목표 스코빌 지수 `K`
- 해야 할 일:
  - 모든 음식의 스코빌 지수가 `K` 이상이 되도록 음식을 섞는다.
  - 한 번 섞을 때마다 가장 맵지 않은 음식 2개를 꺼낸다.
  - 두 음식을 섞어 새 스코빌 지수를 만든 뒤 다시 음식 목록에 넣는다.
  - 모든 음식이 `K` 이상이 되는 최소 섞기 횟수를 구한다.
- 주의할 점:
  - 매번 가장 작은 스코빌 지수 2개를 빠르게 찾아야 한다.
  - 단순 정렬을 반복하면 `scoville`의 길이가 최대 1,000,000이므로 효율성 문제가 발생한다.
  - 음식이 1개만 남았는데도 그 음식의 스코빌 지수가 `K`보다 작다면 더 이상 섞을 수 없으므로 `-1`을 반환해야 한다.
  - 새로 만들어지는 스코빌 지수는 기존 원소보다 커질 수 있으므로 계산 과정에서 값의 범위를 주의해야 한다.
- 반환 형식: 최소 섞기 횟수를 담은 `int`

### 접근법
1. Solution1: PriorityQueue 방식 (권장) 
   - 가장 작은 값 2개를 반복해서 꺼내야 하므로 Min Heap 구조인 `PriorityQueue`를 사용한다.
   - Java의 `PriorityQueue<Integer>`는 기본적으로 오름차순 우선순위 큐이므로 가장 작은 값이 먼저 나온다.
   - 내부적으로는 완전 이진 트리 형태의 Heap 배열을 사용하며, `poll()`을 하면 루트의 최솟값이 빠지고 마지막 노드가 루트로 올라온 뒤 아래로 내려가며 정렬된다.
   - `offer()`를 하면 새 값이 Heap의 마지막 위치에 들어간 뒤 부모와 비교하면서 위로 올라간다.

2. Solution2: 정렬 반복 방식
   - 매번 배열 또는 리스트를 정렬해서 가장 작은 값 2개를 찾는 방식도 가능하다.
   - 하지만 섞을 때마다 정렬을 반복하면 시간복잡도가 커져 `scoville` 길이가 최대 1,000,000인 조건에서는 권장되지 않는다.
   - 따라서 이 문제의 정석 풀이는 `PriorityQueue` 방식이다.

### 코드 설계
1. 실제 코드의 첫 번째 동작
   - `PriorityQueue<Integer>`를 생성한다.
2. 초기화할 변수
   - `PriorityQueue<Integer> pq = new PriorityQueue<>();`
   - `int count = 0;`
   - 배열 `scoville`의 모든 값을 `pq`에 넣는다.
3. 반복문 안에서 할 일
   - `pq.peek()`가 `K`보다 작은 동안 반복한다.
   - 단, `pq.size()`가 2보다 작으면 더 이상 섞을 수 없으므로 `-1`을 반환한다.
   - 가장 맵지 않은 음식 `first`를 꺼낸다.
   - 두 번째로 맵지 않은 음식 `second`를 꺼낸다.
   - 새 스코빌 지수 `mixed = first + second * 2`를 계산한다.
   - 새 음식을 다시 `pq`에 넣는다.
   - 섞기 횟수 `count`를 1 증가시킨다.
4. 반환 형식
   - 모든 음식의 스코빌 지수가 `K` 이상이면 `count`를 반환한다.
   - 불가능하면 반복 중 `-1`을 반환한다.

## 코드 구현

### Solution1: 방식 (권장)
- 핵심 포인트
  - 가장 작은 값 2개를 반복해서 꺼내야 하므로 Min Heap을 사용한다.
  - Java에서는 `PriorityQueue`가 기본적으로 Min Heap처럼 동작한다.
  - `peek()`는 현재 가장 작은 값을 확인한다.
  - `poll()`은 현재 가장 작은 값을 꺼낸다.
  - `offer()`는 새 값을 Heap에 삽입한다.
- 시간복잡도: O(N log N)
  - 초기 삽입에 최대 `N log N`
  - 섞는 과정에서 `poll()` 2번, `offer()` 1번이 반복된다.
  - 각 연산은 Heap 높이에 따라 `O(log N)`이다.
- 공간복잡도: O(N)
  - 모든 스코빌 지수를 PriorityQueue에 저장한다.

### Solution2: 방식
- 핵심 포인트
  - 리스트를 정렬해서 가장 작은 값 2개를 계속 찾는 방식이다.
  - 논리적으로는 가능하지만 매번 정렬이 필요하므로 비효율적이다.
  - 입력 크기가 작다면 이해용으로는 가능하지만, 이 문제의 제한 조건에서는 권장하지 않는다.
- 시간복잡도: O(N<sup>2</sup> log N)
  - 섞을 때마다 정렬을 반복할 수 있기 때문에 효율성이 좋지 않다.
- 공간복잡도: O(N)
