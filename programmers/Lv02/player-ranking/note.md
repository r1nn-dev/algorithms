# Problem: Player Ranking

### 문제
문제: 경기 결과가 주어질 때, 정확하게 순위를 매길 수 있는 선수의 수를 구하세요.
```text
n명의 권투선수가 권투 대회에 참여했고 각각 1번부터 n번까지 번호를 받았습니다.
권투 경기는 1대1 방식으로 진행이 되고, 만약 A 선수가 B 선수보다 실력이 좋다면 A 선수는 B 선수를 항상 이깁니다.
심판은 주어진 경기 결과를 가지고 선수들의 순위를 매기려 합니다.
하지만 몇몇 경기 결과를 분실하여 정확하게 순위를 매길 수 없습니다.

선수의 수 n, 경기 결과를 담은 2차원 배열 results가 매개변수로 주어질 때 정확하게 순위를 매길 수 있는 선수의 수를 return 하도록
solution 함수를 작성해주세요.
```
- 제한사항
  - 선수의 수는 1명 이상 100명 이하입니다.
  - 경기 결과는 1개 이상 4,500개 이하입니다.
  - `results` 배열 각 행 `[A, B]`는 A 선수가 B 선수를 이겼다는 의미입니다.
  - 모든 경기 결과에는 모순이 없습니다.

### 입력
- `int n`: 선수의 수
- `int[][] results`: 2차원 정수 배열
  - 각 원소 [A, B]는 A 선수가 B 선수를 이겼다는 의미이다.
### 출력 
- 정확하게 순위를 매길 수 있는 선수의 수를 정수로 반환한다.
### 입출력 예시
```
예시 1:
입력:
n = 5
results = [[4,3],[4,2],[3,2],[1,2],[2,5]]
출력: return = 3 (1,2,5번 선수가 순위 확정)
```

### 문제 이해
- 입력:
  - `n`: 선수의 수
  - `results`: 경기 결과 배열
    - 각 원소 [A, B]는 A 선수가 B 선수를 이겼다는 의미이다. 
- 해야 할 일:
  - 각 선수에 대해 자신의 상대적 순위를 확정할 수 있는지 판단해야 한다.
  - 어떤 선수의 순위가 확정되려면, 해당 선수를 기준으로 나머지 `n - 1`명의 선수와 승패 관계를 알 수 있어야 한다.
- 주의할 점:
  - 직접 경기한 결과만 보는 것이 아니다.
  - 간접적인 승패 관계도 고려해야 한다.
  - 예를 들어 `A > B`, `B > C`라면 `A > C`도 알 수 있다.
  - 어떤 선수 기준으로 `이길 수 있는 선수 수 + 자신을 이길 수 있는 선수 수 == n - 1`이면 순위가 확정된다.
- 반환 형식: 정확하게 순위를 매길 수 있는 선수의 수를 int로 반환한다.

### 접근법
1. Solution1: DFS 기반 승패 관계 탐색 방식 (권장)
   - 경기 결과를 정방향 그래프와 역방향 그래프로 나눈다.
   - 정방향 그래프에서는 내가 이길 수 있는 선수들을 탐색한다.
   - 역방향 그래프에서는 나를 이길 수 있는 선수들을 탐색한다.
   - 두 탐색 결과의 합이 `n - 1`이면 해당 선수의 순위가 확정된다.
2. Solution2: Floyd-Warshall 기반 승패 관계 갱신 방식
   - `win[a][b] = true`를 A가 B를 이길 수 있다는 의미로 사용한다.
   - `A > B`, `B > C`이면 `A > C`도 true로 갱신한다.
   - 모든 선수 쌍의 승패 관계를 구한 뒤, 각 선수마다 다른 모든 선수와 관계가 있는지 확인한다.
   - 선수 수가 최대 100명이므로 `O(N^3)`도 충분히 가능하다.

### 코드 설계
1. 그래프를 2개 만든다.
   - `winGraph`
     - A -> B
     - A가 이긴 선수들을 저장한다.
   - `loseGraph`
     - B -> A
     - B를 이긴 선수들을 저장한다.
2. results를 순회하면서 그래프를 초기화한다.
   - `[winner, loser]` 형태로 꺼낸다.
   - `winGraph[winner].add(loser)`
   - `loseGraph[loser].add(winner)`
3. 각 선수 i에 대해 DFS를 수행한다.
   - `winCount`: i가 이길 수 있는 선수 수
   - `loseCount`: i를 이길 수 있는 선수 수
4. `winCount + loseCount == n - 1`이면 순위를 정확히 알 수 있다.
   - `answer++`
5. `answer`를 반환한다.


## Solution1: DFS 기반 승패 관계 탐색 

### 코드 설계
1. 그래프를 2개 만든다.
   - `winGraph`
     - `A -> B`: A가 이긴 선수들을 저장한다.
   - `loseGraph`
     - `B -> A`: B를 이긴 선수들을 저장한다.
2. `results`를 순회하면서 그래프를 초기화한다.
   - `[winner, loser]` 형태로 꺼낸다.
   - `winGraph.get(winner).add(loser)`
   - `loseGraph.get(loser).add(winner)`
3. 각 선수 마 DFS를 수행한다.
   - `winCount`: 해당 선수가 이길 수 있는 선수 수
   - `loseCount`: 해당 선수를 이길 수 있는 선수 수
4. `winCount + loseCount == n - 1`이면 순위를 정확히 알 수 있다.
5. 순위가 확정된 선수 수를 반환한다.

### 코드 구현 
- 핵심 포인트
- 시간복잡도: `O(N * (N + M))`
  - 선수 수를 N, 경기 결과 수를 M이라고 할 때 각 선수마다 정방향 DFS와 역방향 DFS를 수행한다.
  - 상수 2는 제거하므로 O(N × (N + M))이다.
- 공간복잡도: `O(N + M)`
  - 그래프 저장에 O(N + M)이 필요하다.
  - DFS 방문 배열에 O(N)이 필요하다.

## Solution2: Floyd-Warshall 기반 승패 관계 갱신 방식

### 코드 설계
1. `boolean[][] win` 배열을 만든다.
   - `win[a][b] == true`는 A 선수가 B 선수를 이길 수 있다는 의미이다.
2. `results`를 순회하면서 직접 승패 관계를 저장한다.
   - `[A, B]`이면 `win[A][B] = true`
3. Floyd-Warshall 방식으로 간접 승패 관계를 갱신한다.
   - `i`가 `k`를 이기고, `k`가 `j`를 이기면 `i`는 `j`를 이길 수 있다.
   - 즉, `win[i][k] && win[k][j]`이면 `win[i][j] = true`
4. 각 선수마다 다른 모든 선수와 관계가 있는지 확인한다.
   - `win[i][j] == true`이면 i가 j보다 강하다.
   - `win[j][i] == true`이면 j가 i보다 강하다.
   - 둘 중 하나라도 true이면 두 선수의 상대적 순위를 알 수 있다.
5. 한 선수 기준으로 비교 가능한 선수 수가 `n - 1`이면 순위가 확정된다.
6. 순위가 확정된 선수 수를 반환한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N^3)`
  - Floyd-Warshall의 3중 반복문 때문에 O(N^3)이다.
  - N <= 100이므로 최대 약 1,000,000번 수준이라 충분히 통과 가능하다.
- 공간복잡도: `O(N^2)`
  - 모든 선수 쌍의 승패 관계를 저장하기 위해 2차원 배열을 사용한다.
