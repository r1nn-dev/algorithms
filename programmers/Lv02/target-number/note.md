# Problem: Target Number

### 문제
문제: 배열 numbers의 각 숫자 앞에 +, - 중 하나를 선택해서 target을 만드는 경우의 수를 구하세요.
```text
n개의 음이 아닌 정수들이 있습니다.
이 정수들을 순서를 바꾸지 않고 적절히 더하거나 빼서 타겟 넘버를 만들려고 합니다.
예를 들어 [1, 1, 1, 1, 1]로 숫자 3을 만들려면 다음 다섯 방법을 쓸 수 있습니다.
    -1+1+1+1+1 = 3
    +1-1+1+1+1 = 3
    +1+1-1+1+1 = 3
    +1+1+1-1+1 = 3
    +1+1+1+1-1 = 3
사용할 수 있는 숫자가 담긴 배열 numbers, 타겟 넘버 target이 매개변수로 주어질 때
숫자를 적절히 더하고 빼서 타겟 넘버를 만드는 방법의 수를 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항:
  - 주어지는 숫자의 개수는 2개 이상 20개 이하 
  - 각 숫자는 1 이상 50 이하인 자연수 
  - 타겟 넘버는 1 이상 1000 이하인 자연수


### 입력
- `int[] numbers`: 부호를 붙일 숫자 배열
- `int target`: 만들어야 하는 목표값
### 출력
- `target`을 만들 수 있는 부호 조합의 개수를 반환한다. 
### 입출력 예시
```
예시 1:
입력:
numbers = [1, 1, 1, 1, 1]
target = 3
출력: return = 5

예시 2:
입력:
numbers = [4, 1, 2, 1]
target = 4
출력: return = 2
```

문제 이해
- 입력:
  - `numbers`: 부호를 붙일 숫자 배열
  - `target: 만들어야 하는 목표값
- 해야 할 일:
  - 각 숫자 앞에 + 또는 -를 붙이는 모든 경우를 탐색한다.
  - 모든 숫자를 사용했을 때 누적합이 target과 같으면 경우의 수를 1 증가시킨다.
- 주의할 점:
  - 숫자를 반드시 모두 사용해야 한다.
  - 각 숫자는 한 번씩만 사용한다.
  - 중간 누적합이 target과 달라도 이후 숫자에 따라 target이 될 수 있으므로 중간에서 함부로 가지치기하지 않는다.
- 반환 형식: `target`을 만들 수 있는 경우의 수를 int로 반환한다.

### 접근법
1. Solution1: DFS 완전 탐색
   - 각 숫자마다 +를 선택하는 경우와 -를 선택하는 경우를 재귀적으로 탐색한다.
   - 모든 숫자를 사용한 시점에 누적합이 target과 같은지 확인한다.

### 코드 설계
1. `solution()`에서 정답 변수 `answer`를 초기화한다.
2. `dfs(numbers, target, depth, sum)`을 호출한다.
   - `depth`: 현재 처리 중인 숫자의 인덱스
   - `sum`: 현재까지 만든 누적합
3. DFS 내부에서 할 일
   - `depth == numbers.length`이면 모든 숫자를 사용한 상태이다.
   - 이때 `sum == target`이면 정답을 1 증가시킨다.
   - 아직 숫자가 남아 있다면 현재 숫자를 더하는 경우와 빼는 경우로 재귀 호출한다.
4. 최종적으로 `answer`를 반환한다.


## Solution1: DFS 완전 탐색 

### 코드 설계
1. `solution()`에서 정답 변수 `answer`를 초기화한다.
2. `dfs(numbers, target, depth, sum)`을 호출한다.
   - `depth`: 현재 처리 중인 숫자의 인덱스
   - `sum`: 현재까지 만든 누적합
3. DFS 내부에서 할 일
   - `depth == numbers.length`이면 모든 숫자를 사용한 상태이다.
   - 이때 `sum == target`이면 정답을 1 증가시킨다.
   - 아직 숫자가 남아 있다면 현재 숫자를 더하는 경우와 빼는 경우로 재귀 호출한다.
4. 최종적으로 `answer`를 반환한다.

### 코드 구현
- 핵심 포인트
    - 각 숫자마다 +, - 두 가지 선택지를 만든다.
    - 재귀 호출의 깊이는 현재까지 사용한 숫자의 개수를 의미한다.
    - 모든 숫자를 사용한 뒤 target과 비교한다.
- 시간복잡도: `O(2^n)`
  - 각 숫자마다 +, - 두 가지 경우가 생기므로 전체 경우의 수는 2^n개이다.
- 공간복잡도: `O(n)`
  - 재귀 호출 깊이가 numbers의 길이만큼 발생한다.
