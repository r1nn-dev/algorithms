# Problem: Travel Route

### 문제
문제: 항공편 tickets이 주어질 때 모든 항공편을 정확히 한 번씩 이용하는 여행 경로를 구하세요. 
```text
주어진 항공권을 모두 이용하여 여행경로를 짜려고 합니다. 항상 "ICN" 공항에서 출발합니다.
항공권 정보가 담긴 2차원 배열 tickets가 매개변수로 주어질 때, 방문하는 공항 경로를 배열에 담아 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 모든 공항은 알파벳 대문자 3글자로 이루어진다.
  - 주어진 공항 수는 3개 이상 10,000개 이하이다.
  - tickets의 각 행 [a, b]는 a 공항에서 b 공항으로 가는 항공권이 있다는 의미이다.
  - 주어진 항공권은 모두 사용해야 한다.
  - 만일 가능한 경로가 2개 이상일 경우 알파벳 순서가 앞서는 경로를 return 한다.
  - 모든 도시를 방문할 수 없는 경우는 주어지지 않는다.

### 입력
- 2차원 문자열 배열 `tickets`
- `tickets[i][0]`: 출발 공항
- `tickets[i][1]`: 도착 공항
### 출력 
- `ICN`에서 출발하여 모든 항공권을 정확히 한 번씩 사용한 여행 경로
- 반환 타입: `String[]`
### 입출력 예시
```
예시 1:
입력:
tickets = [["ICN","JFK"],["HND","IAD"],["JFK","HND"]]
출력: ["ICN", "JFK", "HND", "IAD"]

예시 2:
입력: [["ICN", "SFO"], ["ICN", "ATL"], ["SFO", "ATL"], ["ATL", "ICN"], ["ATL","SFO"]]
출력: ["ICN", "ATL", "ICN", "SFO", "ATL", "SFO"]
```
- ["ICN", "SFO", "ATL", "ICN", "ATL", "SFO"] 순으로 방문할 수도 있다.
- 하지만 ["ICN", "ATL", "ICN", "SFO", "ATL", "SFO"] 가 알파벳 순으로 앞서므로 해당 경로를 반환한다.

### 문제 이해
- 입력:
  - 항공권 정보가 담긴 2차원 문자열 배열 `tickets`
  - 각 항공권은 `tickets[출발 공항, 도착 공항]`
- `ICN` 공항에서 출발해 주어진 항공권을 모두 정확히 한 번씩 사용하여 항공권을 모두 사용했을 때의 공항 방문 순서를 반환한다.
- 같은 공항을 여러 번 방문할 수 있다.
- 방문 체크는 공항이 아니라 항공권 단위로 해야 한다.
- 가능한 경로가 여러 개라면 알파벳 순으로 가장 앞서는 경로를 선택해야 한다.
- 
- 반환 형식:
  - 방문한 공항 경로를 String[]로 반환한다.
  - 항공권이 n개라면 경로에 포함되는 공항 수는 n + 1개이다.

## Solution1: 정렬 + DFS + 백트래킹 방식 
### 코드 설계
1. 항공권 배열 `tickets` 출발 공항 기준 오름차순 정렬
   - 출발 공항이 같으면 도착 공항 기준 오름차순 정렬
2. 항공권 사용 여부를 저장할 visited 배열
   - `visited[i] == true`: i번째 항공권을 이미 사용한 상태.
3. 현재 경로를 저장할 `List<String>`
   - 시작 공항 ICN을 먼저 넣는다.
4. DFS 실행
   - 현재 공항에서 출발할 수 있는 미사용 항공권을 찾는다.
   - 해당 항공권을 사용 처리한다.
   - 도착 공항을 경로에 추가한다.
   - 다음 공항에서 다시 DFS를 수행한다.
   - 항공권을 모두 사용하면 탐색을 종료한다. -> 경로 길이 `tickets.length + 1`
5. 실패한 경로라면 백트래킹한다.
   - 마지막에 추가한 공항을 제거한다.
   - 사용 처리한 항공권을 다시 미사용 상태로 되돌린다.
### 코드 구현
- 핵심 포인트
  - 공항 방문 여부가 아니라 항공권 사용 여부를 체크한다.
  - 항공권을 미리 정렬하면 먼저 완성된 경로가 알파벳 순으로 가장 앞선 경로가 된다.
- 시간복잡도: `O(N! * N)`
  - 백트래킹은 가능한 경로 조합을 탐색할 수 있으므로 최악의 경우 비용이 크다.
- 공간복잡도: `O(N)`
  - visited 배열, 경로 리스트, 재귀 호출 스택이 항공권 수에 비례한다.

### Solution2: 우선순위 큐 + DFS 방식 
### 코드 설계
1. 출발 공항을 key로, 도착 공항 목록을 value로 하는 그래프를 만든다.
2. 도착 공항 목록 `PriorityQueue`
   - 항상 알파벳 순으로 가장 앞서는 도착 공항을 먼저 꺼낼 수 있다.
3. DFS 실행
   - ICN에서 시작한다.
   - 현재 공항에서 갈 수 있는 도착지가 남아 있으면 계속 이동한다. 
   - 더 이상 갈 곳이 없을 때 현재 공항을 경로의 앞쪽에 추가한다.
4. 최종 경로를 `String[]`로 변환해 반환한다. 
### 코드 구현
- 핵심 포인트
  - `poll()`을 호출하면 알파벳 순으로 가장 앞서는 도착지가 나온다.
  - 재귀가 끝나는 시점에 현재 공항을 경로 앞쪽에 넣는다.
- 시간복잡도: `O(N log N)`
  - 각 항공권을 우선순위 큐에 넣고 꺼내는 비용
- 공간복잡도: `O(N)
  - 그래프, 경로, 재귀 호출 스택이 항공권 수에 비례한다.
