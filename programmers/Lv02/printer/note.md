# Problem: Printer

문제: 프린터 큐에서 중요도가 높은 문서부터 인쇄됩니다. location 위치의 문서가 몇 번째로 인쇄되는지 구하라.
```text
운영체제의 역할 중 하나는 컴퓨터 시스템의 자원을 효율적으로 관리하는 것입니다.
이 문제에서는 운영체제가 다음 규칙에 따라 프로세스를 관리할 경우 특정 프로세스가 몇 번째로 실행되는지 알아내면 됩니다.
  1. 실행 대기 큐(Queue)에서 대기중인 프로세스 하나를 꺼냅니다.
  2. 큐에 대기중인 프로세스 중 우선순위가 더 높은 프로세스가 있다면 방금 꺼낸 프로세스를 다시 큐에 넣습니다.
  3. 만약 그런 프로세스가 없다면 방금 꺼낸 프로세스를 실행합니다.
    3.1 한 번 실행한 프로세스는 다시 큐에 넣지 않고 그대로 종료됩니다.
예를 들어 프로세스 4개 [A, B, C, D]가 순서대로 실행 대기 큐에 들어있고, 우선순위가 [2, 1, 3, 2]라면 [C, D, A, B] 순으로 실행하게 됩니다.
현재 실행 대기 큐(Queue)에 있는 프로세스의 중요도가 순서대로 담긴 배열 priorities와, 몇 번째로 실행되는지 알고싶은 프로세스의 위치를 알려주는 location이 매개변수로 주어질 때,
해당 프로세스가 몇 번째로 실행되는지 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - priorities의 길이는 1 이상 100 이하이다.
  - priorities의 원소는 1 이상 9 이하의 정수이다.
  - priorities의 원소는 우선순위를 나타내며 숫자가 클수록 우선순위가 높다.
  - location은 0 이상 (대기 큐에 있는 프로세스 수 - 1) 이하의 값을 가진다.
  - priorities의 가장 앞에 있으면 0, 두 번째에 있으면 1 … 과 같이 표현다.

### 입력
- 정수 배열 priorities
- 찾고 싶은 문서의 위치 location
### 출력 
- location 위치의 문서가 몇 번째로 인쇄되는지 나타내는 정수
### 입출력 예시
```
예시 1:
입력:
priorities = [2, 1, 3, 2],
location = 2
출력: 1 (위치 2의 문서(3)가 1번째 인쇄)

예시 2:
입력:
priorities = [1, 1, 9, 1, 1, 1],
location = 0
출력: 5
```

### 문제 이해
- 입력: 문서들의 중요도가 담긴 정수 배열 priorities와 찾고 싶은 문서의 위치 location이 주어진다.
- 프린터는 큐의 맨 앞 문서를 먼저 확인한다.
- 맨 앞 문서보다 중요도가 높은 문서가 큐 안에 있으면, 해당 문서는 인쇄되지 않고 큐의 맨 뒤로 이동한다.
- 맨 앞 문서보다 중요도가 높은 문서가 없으면, 해당 문서는 인쇄되고 큐에서 제거된다.
- 이 과정을 반복하면서 location 위치에 있던 문서가 몇 번째로 인쇄되는지 구한다.
- 반환 형식: location 위치의 문서가 인쇄되는 순서를 int형으로 반환한다.

### 접근법
1. 방법1: Queue에 다 저장하는 방식 (권장)
   - 각 문서를 [우선순위, 원래 위치] 형태로 큐에 저장한다.
   - 큐에서 문서를 하나 꺼낸 뒤, 큐 안에 현재 문서보다 우선순위가 높은 문서가 있는지 확인한다.
     - 더 높은 우선순위가 있으면 현재 문서를 큐의 뒤에 다시 넣는다.
     - 더 높은 우선순위가 없으면 현재 문서를 인쇄하고 출력 순서를 1 증가시킨다.
   - 인쇄한 문서의 원래 위치가 location이면 현재 출력 순서를 반환한다.
2. 방법2: Queue + 우선순위 개수 배열 사용하는 방식
   - 우선순위는 1부터 9까지만 존재하므로 int[] count 배열로 각 우선순위의 개수를 저장한다.
   - 큐에서 꺼낸 문서의 우선순위가 현재 최대 우선순위보다 낮으면 다시 큐에 넣는다.
   - 현재 최대 우선순위와 같으면 인쇄한다.
   - 한계
     - 매번 큐 전체를 순회하지 않아도 되므로 우선순위 범위가 작을 때 효율적이다.

## 코드 구현
### Solution1: Queue에 다 저장하는 방식 (권장)
- 핵심 포인트
- 시간복잡도: `O(N^2)`
  - 최대 N개의 문서를 처리하고, 각 처리 과정에서 큐 전체 확인 가능 
  - N <= 100이므로 충분히 통과 가능하다.
- 공간복잡도: `O(N)`

### Solution2: Queue + 우선순위 개수 배열 사용하는 방식
- 핵심 포인트
  - 우선순위는 1 ~ 9
  - `count[priority]`: 해당 우선순위를 가진 문서 개수
  - 큐 전체를 매번 순회하지 않아도 된다
- 시간복잡도: O(N * 9) → `O(N)`
  - 현재 최대 우선순위를 찾을 때 최대 9칸만 확인한다.
- 공간복잡도: O(N + 10) → `O(N)`
