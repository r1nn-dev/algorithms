# Problem: Carpet

### 문제
문제: 카펫의 갈색과 노란색 격자 수가 주어질 때, 카펫의 크기를 구하세요.
```text
아래와 같이 중앙에는 노란색으로 칠해져 있고 테두리 1줄은 갈색으로 칠해져 있는 격자 모양 카펫이 있습니다.
    갈 갈 갈 갈
    갈 노 노 갈
    갈 갈 갈 갈
카펫에서 갈색 격자의 수 brown, 노란색 격자의 수 yellow가 매개변수로 주어질 때
카펫의 가로, 세로 크기를 순서대로 배열에 담아 return 하도록 solution 함수를 작성해주세요.
```
- 제한사항
  - 갈색 격자의 수 brown은 8 이상 5,000 이하인 자연수이다. 
  - 노란색 격자의 수 yellow는 1 이상 2,000,000 이하인 자연수이다. 
  - 카펫의 가로 길이는 세로 길이와 같거나, 세로 길이보다 길다. 

### 입력
- brown: 갈색 격자의 개수
- yellow: 노란색 격자의 개수
### 출력 
- [가로 길이, 세로 길이]
### 입출력 예시
```
예시 1:
입력: brown = 10, yellow = 2
출력: [4, 3]

예시 2:
입력: brown = 8, yellow = 1
출력: [3, 3]

예시 3:
입력: brown = 24, yellow = 24
출력: [8, 6]
```


## 풀이

### 문제 정보
- 플랫폼: 프로그래머스 
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42842
- 난이도: Lv 02 
- 주제: 완전탐색, 조건 만족 
- 문제 핵심:
  - 갈색 = (w-2)*(h-2) = yellow
  - 전체 = w*h = brown + yellow
  - 가능한 w, h 찾기
- 최초 풀이: 260526
- 수정 버전: 260615

### 문제 이해
- 입력:
    - brown: 테두리에 있는 갈색 격자의 개수
    - yellow: 중앙에 있는 노란색 격자의 개수 
- 해야 할 일: 전체 카펫의 가로 길이와 세로 길이를 구한다.
- 주의할 점:
    - 가로 길이는 세로 길이보다 크거나 같아야 한다.
    - 노란색 영역은 반드시 갈색 테두리 안쪽에 있어야 한다.
    - 따라서 실제 노란색 영역의 크기는 (가로 - 2) * (세로 - 2)이다.
- 반환 형식: [가로, 세로] 형태의 int[]

### 접근법
1. Solution1: 약수 쌍 완전탐색 방식 (권장)
   - 전체 격자 수 brown + yellow의 약수 쌍을 찾고, 그중 내부 영역이 yellow와 일치하는 가로와 세로를 찾는다.

### 코드 설계
1. 전체 격자 수 total = brown + yellow를 구한다.
2. 세로 길이 후보 h를 3부터 탐색한다.
   - 카펫은 최소한 테두리 1줄과 내부 노란색 영역이 있어야 하므로 최소 높이는 3이다.
3. `total % h == 0`이면 h는 세로 후보가 될 수 있다.
    - 이때 가로 길이 `w = total / h`이다.
4. `w >= h` 조건을 만족하는지 확인한다.
5. 내부 노란색 영역 `(w - 2) * (h - 2)`가 yellow와 같으면 정답이다.
6. [w, h]를 반환한다.

## 코드 구현

### Solution1: 약수 쌍 완전탐색 방식 (권장)
- 핵심 포인트
    - 카펫의 전체 넓이는 brown + yellow이다.
    - 가능한 가로와 세로는 전체 넓이의 약수 쌍이다.
    - 내부 영역 (w - 2) * (h - 2)가 yellow와 같으면 정답이다.
- 시간복잡도: `O(√N)`
    - N = brown + yellow
    - 세로 후보를 sqrt(total)까지만 탐색한다.
- 공간복잡도: `O(1)`
