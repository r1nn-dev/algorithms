# Problem: Parentheses Split 
 
문제 요약: 괄호 문자열을 두 개의 연속한 문자열로 나눴을 때, 두 문자열이 모두 올바른 괄호 문자열이 되는 분할 위치의 개수를 구하라. 

## 풀이

### 문제 정보
- 난이도: Lv 02 정도 (기준은 프로그래머스)
- 주제: 스택, 괄호 문자열, 누적합, 그리디
- 문제 핵심 
  - 괄호 균형값이 `0`이 되는 지점 카운트 
  - 전체 문자열이 올바른 괄호 문자열인지 검증 
- 최초 풀이: 260626  
- 수정 버전: 260708 

### 접근법
1. Solution1: 괄호 균형값 카운팅 
   - `(`는 `+1`, `)`는 `-1`로 누적하면서 전체 유효성을 검증하고, 중간에 균형값이 `0`이 되는 위치를 센다.


## Solution1: 방식 (권장)
### 코드 설계
1. N과 괄호 문자열 S를 입력받는다. 
2. `balance`를 0으로 초기화한다.
3. 문자열을 왼쪽부터 순회한다.
   - `(`이면 `balance++` 
   - `)`이면 `balance--`
4. 순회 중 `balance < 0`이 되면 올바른 괄호 문자열이 될 수 없으므로 `isValid = false`로 표시한다.
5. 마지막 위치가 아닌데 `balance == 0`이면 분할 가능한 위치이므로 `answer++` 한다.
6. 순회가 끝난 뒤 `isValid == false`이거나 `balance != 0`이면 0을 출력한다.
7. 그렇지 않으면 `answer`를 출력한다.

### 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N)`
- 공간복잡도: `O(1)`
