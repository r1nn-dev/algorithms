# Problem: Nearest Point Query
 
문제 요약: 1차원 수직선 위의 점들과 질의 좌표가 주어졌을 때, 각 질의 좌표에서 가장 가까운 점의 좌표를 출력하라. 

## 풀이

### 문제 정보
- 난이도: Lv 02 정도 (기준은 프로그래머스)
- 주제: 정렬, 이분 탐색, lower bound
- 문제 핵심:
  - 정렬 후 이분 탐색
  - 양쪽 후보 비교 
- 최초 풀이: 260527 
- 수정 버전: 260705 

### 접근법
1. Solution1: 정렬 + 이분 탐색 lower bound 방식 
   - 점의 좌표를 정렬한 뒤, 각 질의마다 `lower_bound`로 `p` 이상인 첫 번째 위치를 찾고, 그 위치와 바로 왼쪽 위치만 비교한다.


## Solution1: 정렬 + 이분 탐색 lower bound 방식 
### 코드 설계
1. 점의 개수 N, 질의 개수 Q를 입력받는다.
2. 점의 좌표를 `long[] points`에 저장한다.
   - 좌표 범위가 `-10^18 ~ 10^18`이므로 int가 아니라 long을 사용해야 한다.
3. 점의 좌표 배열을 오름차순 정렬한다.
4. 각 질의 `p`에 대해 이분 탐색으로 `p` 이상인 첫 번째 인덱스를 찾는다.
   - `idx == 0`이면 가장 왼쪽 점이 정답이다.
   - `idx == N`이면 가장 오른쪽 점이 정답이다.
   - 그 외에는 `points[idx - 1]`과 `points[idx]` 중 더 가까운 점을 고른다.
   - 거리가 같으면 좌표가 더 작은 `points[idx - 1]`을 선택한다.
5. 반환 형식: 각 질의의 정답을 `StringBuilder`에 누적한 뒤 한 번에 출력한다.

## 코드 구현
- 핵심 포인트
- 시간복잡도: `O(N log N + Q log N)`
  - 정렬: `O(N log N)`
  - 질의 처리: `O(Q log N)`
- 공간복잡도: `O(N)`
  - 점 좌표 배열
